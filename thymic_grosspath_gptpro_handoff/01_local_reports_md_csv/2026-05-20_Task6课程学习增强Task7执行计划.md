# Task6 课程学习增强 Task7 执行计划

日期：2026-05-20

## 一、核心目标

我们这轮不把六分类当成单独展示结果，而是把六分类当成更细粒度的监督信号，观察它能否反过来提升医生最关心的两分法结果。

最终判断标准仍然放在 Task7：

- 低危组：A、AB、B1
- 高危组：B2、B3、TC
- 主指标：Accuracy、Balanced Accuracy、F1、AUC
- 重点安全指标：高危打成低危的 FN 数量
- 重点困难集指标：43 例持续 hard-core 病例中能救回多少例

当前锁定对照线：

- 稳定主结果：Stage2 + Stage3 fold-wise blend，Accuracy 0.7649，Balanced Accuracy 0.7649，F1 0.7633，AUC 0.8117，FN 33。
- 候选上限结果：balcore blend，Accuracy 0.7684，Balanced Accuracy 0.7685，F1 0.7676，AUC 0.8154，FN 32，但稳定性弱于主结果。

这轮实验只有超过上述结果，或者明显减少 FN / 救回 hard-core 病例，才算对主线有价值。

## 二、为什么要做 Task6 课程学习

直接二分类只告诉模型“低危 / 高危”，但六分类会让模型看到更细的边界：

- A、AB、B1 都是低危，但外观差异并不完全一样。
- B2、B3、TC 都是高危，但模型容易把 B2/B3/TC 与 AB/B1 混在一起。
- 现在最难的问题不是普通相邻错误，而是跨风险等级错误，尤其是低危升高危和高危降低危。

所以这轮的思路是：

- 先让模型按六分类学更细的分型结构。
- 再把六分类概率折叠回 Task7，看它对低危 / 高危边界有没有帮助。
- 如果单独六分类折叠不够强，再和现有 Task7 主模型做 fold-wise 融合。

## 三、实验设计

### 1. 固定公平数据设置

全部实验使用同一套当前数据和同一套 5-fold：

- Registry：`outputs/batch1_batch2_task567_20260514/frozen_inputs/combined_task567_registry.csv`
- Split：`outputs/batch1_batch2_task567_20260514/frozen_inputs/combined_5fold_assignments.csv`
- Image root：`outputs/batch1_batch2_task567_20260514/frozen_inputs/selected_images`

规则：

- 所有模型必须 5-fold OOF。
- 每一折只用训练折训练，用验证折选参数，用测试折出结果。
- 不允许根据测试折调阈值。
- 所有结果保留 case-level OOF，后续混淆矩阵和医生复核都从 OOF 复原。

### 2. 先生成 Task6 教师模型

先跑一组 Task6 六分类教师模型，用它们的 OOF 概率给病例分难度。

优先教师：

- DINOv2 vits14 + vitb14 concat，case-level MLP。
- DINOv2 vitb14，case-level MLP。
- DINOv2 vitl14，case-level MLP。

如果 vitl14 权重或显存不稳定，则替换为 vits14 或已有最高 Task6 OOF。

教师模型只用于分 easy / medium / hard，不直接作为最终结论。这样可以避免单一模型的偶然偏差。

### 3. Task6 难度分层

用三个教师的 OOF 结果给每个病例分层：

- `easy`：至少 3 个教师都判断正确，预测类别一致，真类平均概率足够高。
- `medium`：至少 2 个教师判断正确，真类平均概率中等。
- `hard_salvage_teacher`：整体属于 hard，但至少有 1 个教师判断正确，说明仍可能被救回。
- `hard_core`：多个教师都不稳定或持续错误，先不强行当作普通训练样本。

Task6 因为是六分类，概率天然比二分类低，所以阈值会比 Task7 低：

- easy：`correct_count >= 3`，`mean_true_prob >= 0.50`，`min_true_prob >= 0.30`
- medium：`correct_count >= 2`，`mean_true_prob >= 0.30`
- hard salvage：`correct_count >= 1`，`mean_true_prob >= 0.18`

第一轮先用这组宽松阈值，避免六分类下 easy 数量过少。

### 4. Task6 课程训练

采用三阶段：

- Stage1：只训练 easy，让模型先学稳定分型。
- Stage2：加入 medium，让模型学习边界病例。
- Stage3：加入 hard_salvage_teacher，但降低权重，避免 hard 样本把模型带偏。

第一轮不把 `hard_core` 强行放进主训练。hard-core 的用途是复核和后续定向改造，不是直接混入训练集。

第一轮模型：

- Backbone：DINOv2 vits14 + vitb14 concat frozen features。
- Head：case-level MLP。
- Selection metric：Task6 macro-F1。
- Loss：六分类交叉熵 + class balance。
- Stage3 hard weight：0.40 到 0.60 之间做小网格。

### 5. 折叠回 Task7

Task6 输出六个概率：

- `P(A), P(AB), P(B1), P(B2), P(B3), P(TC)`

折叠成 Task7：

- 低危概率：`P(A) + P(AB) + P(B1)`
- 高危概率：`P(B2) + P(B3) + P(TC)`
- 默认阈值：0.50

然后和现有 Task7 主模型做同口径比较：

- Overall Task7 指标。
- 每折 Task7 指标。
- 混淆矩阵。
- FN 病例号。
- hard-core 43 例救回情况。

### 6. 与现有 Task7 主模型融合

如果 Task6 折叠结果本身不超过 Task7 主模型，也不直接否定它。关键看它有没有互补性。

融合方式：

- A 模型：现有 Task7 主模型 `Stage2 + Stage3 fold-wise blend`。
- B 模型：Task6 curriculum 折叠后的高危概率。
- 每一折只在验证集上选择融合权重和阈值。
- 测试折只应用验证折选出的权重和阈值。

重点观察：

- Overall Acc / BAcc / F1 / AUC 是否超过 0.7649。
- FN 是否少于 33。
- 43 个持续 hard-core 里是否能救回一批。
- 是否用 FP 明显增加换取 FN 降低，需要单独报告给医生判断取舍。

## 六、判断规则

第一轮完成后按下面顺序判断：

1. 如果 Task6 curriculum 折叠后直接超过 Task7 主模型，进入复跑确认。
2. 如果 Task6 curriculum 单独不超过，但融合后超过 Task7 主模型，也保留为主线候选。
3. 如果 overall 不超过，但 FN 明显减少，需要单独作为“偏安全策略”给医生评估。
4. 如果 overall、FN、hard-core rescue 都没有改善，则记录为负结果，说明六分类细监督在当前图像条件下没有直接转化成二分类收益。
