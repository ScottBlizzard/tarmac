# 胸腺影像分析阶段性总结与扩库建议
日期：2026-05-07

## 1. 这份材料想说明什么

这份材料面向两个对象：一类是更关注项目方向、结果是否可靠的导师；另一类是更关注任务定义是否符合临床逻辑、结果有没有医学解释空间的医生。我们希望用同一份简报，把这段时间从头到尾做了什么、哪些方法有效、哪些方法无效、目前已经得到什么结果、为什么现在最需要尽快扩展数据集，讲清楚。

一句话概括当前状态：团队已经证明胸腺大体图中存在真实、可学习的分类和分层信号，而且已经把模型主线从旧的复现路线稳定切换到更强的表征模型；目前最大的限制已经不再是代码技巧，而是病例数量和病例多样性。

## 2. 这段时间我们具体完成了哪些工作

整个工作可以概括成四步。

第一步，固定正式数据底座和评估协议。当前项目统一基于 `160` 例病例、`172` 张图像、`12` 例多图病例，全部实验都采用 `5-fold patient-level` 交叉验证，并统一到病例级 `case-level mean probability` 口径下比较。

第二步，把已有思路尽量做透。我们完整复现并迁移了 `SuRImage` 这条 `coarse-to-fine` 线，也依次验证了 `crop` 优化、checkpoint 选择、阈值分析、类别不平衡处理、ordinal loss、轻量融合等多个分支。这个阶段的价值不只是得到一些数字，更重要的是把很多“看起来可能有效”的方向正式试明白，知道哪些值得继续，哪些已经可以停。

第三步，切换到更强的表征主线。我们接入了 `DINOv2`、`PLIP`、`BiomedCoOp`、`INTR` 等外部模型，并优先用冻结特征探针做公平比较。结果很清楚：当前项目的主要瓶颈很大程度上确实在表征能力，而不只是训练细节。

第四步，按医生提出的思路新开任务线。医生提出先做：

- `A/AB`
- `B1/B2/B3`
- `TC`

三分类，再进一步做：

- `A`
- `AB`
- `B1`
- `B2`
- `B3`
- `TC`

六分类。现在这两条线已经不是计划，而是已经完成了首轮正式实验，并拿到了可比较的结果。

## 3. 当前任务体系和目前最好结果

### 3.1 旧任务体系

目前已经完成和保留的任务包括：

- `Task1`：良性胸腺增生 vs 胸腺上皮肿瘤
- `Task2`：胸腺瘤 vs 胸腺癌
- `Task3`：低危胸腺瘤 vs 高危胸腺瘤
- `Task4`：`A / AB / B1 / B2 / B3`

### 3.2 新任务体系

在医生建议基础上新增：

- `Task5`：`A/AB` vs `B1/B2/B3` vs `TC`
- `Task6`：`A / AB / B1 / B2 / B3 / TC`

这里特别说明一下：当前数据原始底座一共有 `8` 类，其中还包括：

- 良性胸腺增生
- 微结节性胸腺瘤

但 `Task5` 和 `Task6` 这两条新线是“胸腺上皮肿瘤谱系”任务，所以没有把良性增生和微结节性胸腺瘤塞进主结果里。这样做不是漏掉数据，而是为了让任务定义更符合临床逻辑，也更方便解释。

### 3.3 当前关键结果

如果只看当前最值得汇报的结果，可以概括成下面几条：

- `Task1`
  - `AUC` 最好：`DINOv2 vitl14 + whole`
  - `AUC = 0.8707`
  - 如果更看重平衡性，`vits14 + whole` 的 `BACC = 0.7607` 更稳

- `Task2`
  - `DINOv2 vits14 + whole`
  - `AUC = 0.8638`
  - `BACC = 0.7875`

- `Task3`
  - `DINOv2 vitl14 + whole + fold-wise threshold`
  - `AUC = 0.7621`
  - `BACC = 0.7167`

- `Task4`
  - `F1` 最好：`DINOv2 vits14 + vitb14`
  - `Macro-F1 = 0.5236`
  - `Macro-AUC = 0.7533`
  - `AUC` 最好：`DINOv2 vits14 + vitb14 + PLIP`
  - `Macro-F1 = 0.5226`
  - `Macro-AUC = 0.7656`

- `Task5`
  - `F1` 最好：`DINOv2 vitl14 + whole`
  - `Macro-F1 = 0.6614`
  - `Macro-AUC = 0.7959`
  - `AUC` 最好：`DINOv2 vitl14 + vitb14` 特征拼接
  - `Macro-F1 = 0.6573`
  - `Macro-AUC = 0.8229`

- `Task6`
  - 当前最好：`DINOv2 (vits14 + vitb14) 特征拼接 + Task5 vitl14 层级先验`
  - `Macro-F1 = 0.5127`
  - `Macro-AUC = 0.8095`

## 4. 我们试了哪些方法，哪些有效，哪些无效

### 4.1 已经被证明有效的方法

到目前为止，效果最明确的方向有四类。

第一类是更强的视觉表征。`DINOv2` 是当前最明确的正向结果来源，也是把项目从“旧主线边际收益变小”带到“重新获得明显提升”的关键。

第二类是 `whole-image` 路线。无论是 `Task3`、`Task4` 还是后面新开的 `Task5/Task6`，当前最强结果基本都不是来自 `crop` 或 `whole+crop`，而是来自 `whole` 或以 `whole` 为主的强表征组合。这意味着对这批胸腺大体图而言，整体形态信息目前比局部裁剪更稳定。

第三类是少量但有效的融合。对 `Task4` 来说，`vits14 + vitb14` 和 `vits14 + vitb14 + PLIP` 确实进一步推高了结果；对 `Task6` 来说，真正起作用的是双 backbone 特征拼接，而不是简单概率加权。

第四类是层级先验。医生建议的三分类不是只在定义上有意义，它还可以反过来作为六分类的组级先验，帮助压缩跨粗分组的大错。这一点在 `Task6` 的增强版上已经体现出来。

### 4.2 已经被证明暂时不值得继续的方向

下面这些方向，我们已经认真试过，但当前不建议继续作为主线投入：

- `PLIP` 单模
- `crop` 或 `whole+crop` 继续升主线
- 旧的 `SuRImage Fusion` 线
- `CONDOR` / ordinal loss
- `stacking`
- `class-wise alpha blend`
- `MLP fusion head`
- `DINO` 轻量微调
- `Task6` 的简单两模概率融合

这些结果同样重要，因为它说明后面如果继续把主要时间投入到这些分支，性价比已经不高。

## 5. 现在 Task5 和 Task6 做到了什么程度

### 5.1 Task5 三分类

`Task5` 当前已经形成了一条比较成熟的临床友好结果线。它回答的是一个很直观的问题：如果按 `A/AB`、`B1-3`、`TC` 这三个层级来分，胸腺大体图能不能支持稳定区分。

现在答案是：可以，而且结果已经比较可观。

最好 `F1` 的结果是：

- `Macro-F1 = 0.6614`
- `Macro-AUC = 0.7959`

这说明：

- 三分类任务不是只能勉强跑通，而是已经能给出较稳定的区分结果
- 对导师来说，这说明临床友好的新任务线已经站住了
- 对医生来说，这说明模型已经不是只会“粗略看图”，而是在这条更符合临床语言的分组下也确实学到了有效信号

### 5.2 Task6 六分类

`Task6` 难度明显更高，因为它既要把 `A` 和 `AB` 分开，也要把 `B1/B2/B3` 分开，同时还要保留 `TC`。

初始最好单模是：

- `Macro-F1 = 0.4864`
- `Macro-AUC = 0.7993`

进一步做了双 backbone 特征拼接和层级先验之后，当前最好结果是：

- `Macro-F1 = 0.5127`
- `Macro-AUC = 0.8095`

这说明：

- 六分类不是做不动
- 也不是原地踏步
- 团队确实把它从不到 `0.49` 推到了 `0.51+`

虽然这个任务还没有像 `Task5` 那样稳，但它已经进入了“可正式汇报”的状态，而不再只是一个计划中的探索方向。

## 6. 怎么看这些数字：AUC、F1、Balanced Accuracy 到底是什么意思

这一部分对导师和医生都很重要，因为不同任务里的数字不能直接横着比。

### 6.1 AUC 是什么

`AUC` 可以理解为模型“排序能力”的指标。它不依赖某一个固定阈值，而是看模型总体上能不能把高风险排在低风险前面、把正确类别排得更靠前。

直观理解：

- `0.5` 附近：接近随机
- `0.6-0.7`：说明有一定信号，但还比较弱
- `0.7-0.8`：说明有比较明确的判别能力
- `0.8+`：在当前这种内部小样本研究里，通常已经算比较强

所以像：

- `Task2 AUC = 0.8638`
- `Task5 AUC = 0.7959`
- `Task6 AUC = 0.8095`

都说明模型里已经有真实、可用的区分信号。

### 6.2 Macro-F1 是什么

`Macro-F1` 是多分类任务里非常重要的指标。它不是简单看“总共对了多少”，而是把每一类都单独看准确率和召回率，再平均。

它的好处是：

- 不会因为某个类别样本多，就把整体分数拉高
- 更适合看每个类别是否都被公平对待

在我们这里：

- `Task5` 是三分类
- `Task6` 是六分类

这两个任务的 `Macro-F1` 都不该拿去和二分类 `AUC` 直接比较，也不该拿去和别的论文里更大数据集、更容易任务的 `80%`、`90%` 直接横比。

### 6.3 Balanced Accuracy 是什么

`Balanced Accuracy` 可以理解为“各类召回率的平均值”。它特别适合看：

- 类别不平衡时，模型是不是只偏向样本多的那一边
- 风险分层任务里，模型是不是两边都照顾到了

比如 `Task3` 的：

- `BACC = 0.7167`

就说明这条风险分层结果不是只靠偏向多数类撑起来的。

## 7. 什么样的数字在当前项目里算可观

这里需要很明确地说：当前项目的数据规模和任务难度，不允许我们用“越接近 1 越好”这样非常粗糙的标准去看结果。

更合理的判断应该按任务类型来分。

### 7.1 二分类任务

在当前这种内部小样本医学影像研究里：

- `AUC > 0.8` 可以认为已经比较强
- `AUC 0.75-0.8` 可以认为是有明显临床研究价值

所以 `Task1` 和 `Task2` 当前结果已经是比较可观的。

### 7.2 三分类任务

对 `Task5` 这种 `120` 例、三分类任务来说：

- `Macro-F1` 到 `0.60` 左右已经算比较像样
- `0.65+` 可以认为是比较好的结果

因此 `Task5` 的：

- `Macro-F1 = 0.6614`

是完全可以认真汇报的。

### 7.3 六分类任务

对 `Task6` 这种每类只有 `20` 例、总共 `6` 类的任务来说：

- `Macro-F1` 到 `0.45` 说明方向成立
- `0.50+` 可以认为是比较可观的结果
- 如果将来能到 `0.60` 左右，那会是非常漂亮的结果

因此当前：

- `Task6 Macro-F1 = 0.5127`

并不是“只有 51 分所以很差”，而是在当前任务难度下已经属于有意义、有提升、可汇报的结果。

需要特别强调的是：这些数字说明的是“研究阶段模型能力”，不是“已经可以直接临床部署”的标准。

## 8. 当前错误分析说明了什么

### 8.1 Task5 的主要短板

`Task5` 的主要问题已经非常集中：

- `TC recall = 0.50`

也就是 `20` 例 `TC` 里只识别对了 `10` 例。

最主要的错误方向是：

- `TC -> B1-3`
- `B1-3 -> TC`

这说明 `Task5` 里最难的不是 `A/AB` 和 `B1-3` 的大方向，而是：

- `TC` 和高危 B 组之间仍然存在视觉重叠

### 8.2 Task6 的主要短板

`Task6` 里最难的类很清楚：

- `A`
- `B2`

其中：

- `A recall = 0.30`
- `B2 recall = 0.35`

最常见的混淆方向是：

- `A -> AB`
- `B1 -> B2`
- `B2 -> B3`
- `B2 -> TC`
- `B3 -> B2`
- `TC -> B2`

这说明：

- `A` 和 `AB` 边界不算清楚
- `B2` 是整个六分类里最像“边界聚集点”的类别
- `B2/B3/TC` 这条带最复杂，也最容易出错

### 8.3 增强版到底改进了什么

和当前最好单模相比，`Task6` 最终增强版不只是把总分从 `0.4864` 推到了 `0.5127`，更重要的是：

- 把跨粗分组的大错从 `39` 个压到了 `34` 个

也就是说，这轮改进不是只在“调一个分数”，而是在帮助模型把一部分明显不合理的大错拉回更合理的范围。

## 9. 为什么现在最大的瓶颈仍然是数据集数量

当前虽然有 `160` 例、`172` 张图，但如果做 `Task6` 六分类，每一类还是只有 `20` 例。放到 `5-fold` 交叉验证里，单折训练时每类实际可用病例很有限。

这会带来三个直接后果：

1. 边界类别学不稳
2. 少数高置信错例会显著影响整体结果
3. 很多小改动即便有效，提升也会很快遇到天花板

因此当前数据已经足够回答：

- 这条方向值不值得做
- 哪条模型主线更有希望
- 哪些方法不值得继续

但还不太足够回答：

- 能不能把六分类做得更高、更稳、更接近临床可解释水平
- 能不能让最难的边界类不再明显受少量病例限制

更重要的是，真正决定提升的不是图像张数本身，而是独立病例数量和病例多样性。如果后续 `800+` 图像对应的是更多新的独立病例、更丰富的边界样本、更完整的胸腺癌与高危 B 组覆盖，那么模型上限还有明显提升空间。

## 10. 现在最需要什么

从当前结果倒推，下一阶段最需要的不是继续在很多小技巧上反复试，而是尽快扩展数据集，尤其优先补下面几类病例：

- `TC`
- `A`
- `B2`
- `B3`
- 以及 `A/AB`、`B1/B2`、`B2/B3`、`B2/TC` 这些边界病例

如果后续扩库能优先补这些位置，那么当前已经建立好的模型主线会很快转化成更强、更稳的新结果。

## 11. 一句话总结

如果用一句话向导师和医生同时汇报当前情况，那么最准确的表述是：

团队已经证明胸腺大体图中存在真实、可学习的分类与分层信号，也已经把模型主线从旧复现路线稳定切换到了更强的表征模型；现在不仅完成了原有任务，还完成了医生建议的三分类和六分类新任务线，并在当前小样本条件下取得了有意义的结果。下一阶段最迫切、最值得投入的任务，仍然是尽快扩展病例数量，特别是补足最难边界带上的新病例。
