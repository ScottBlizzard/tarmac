# Task7 DINO 升级路线计划 v4

时间：2026-05-16

一、当前结论

目前在新数据 Task7 上，普通预训练 backbone 的 whole-image smoke 已经说明：
- ConvNeXt-Large 全学不如 DINO 全学
- Swin-Base 全学明显不如 DINO 全学
- EfficientNet-B3 全学大概率也难超过 DINO 全学

因此后续主线不再横向试普通 backbone，而是回到 DINO 主线上继续做更深升级。

二、后续最值得的 5 条路线

1. 全学单-backbone DINO 强化版
- 目标：验证单 backbone 的 DINO fine-tune，能否逼近或超过当前 frozen concat 基线
- 首选：vitb14
- 训练：Task7、whole-image、last2 blocks + MLP、较大输入尺寸
- 先做 fold1 smoke，再决定是否扩 5-fold

2. 全学两阶段 DINO
- 第一阶段：全学 Task7
- 第二阶段：在 cut-oriented 子集上继续精修
- 目标：保留全局表征稳定性，同时给 cut 信息更多权重

3. Task6 + Task7 联合监督 DINO
- 同一个 backbone 同时学 6 分类和 2 分类
- 目标：用更细监督帮助二分类边界

4. DINO + 结构化特征融合
- 后续医生如果给尺寸等信息，做 image + structure late fusion
- 结构化变量优先级：尺寸 > 视图标签 > 经验标签

5. 多图病例的 DINO 专门聚合
- 不是统一图2，而是按病例级 learned / heuristic 聚合
- 这一条留在主模型站稳后再做

三、执行顺序

P0：停止当前价值不高的 full-view timm smoke，释放 GPU
P1：直接起 DINO vitb14 full-view Task7 强化版 fold1 smoke
P2：如果 fold1 明显正向，再扩完整 5-fold
P3：如果仍不够，再切到两阶段 full->cut 精修

四、当前立即执行项

- 停掉 task7_full_timm_smokes_offline
- 启动 DINO vitb14 full-view Task7 fold1 smoke
- 配置：
  - model = dinov2_vitb14
  - tune_scope = last2_blocks
  - head_type = mlp
  - image_size = 392
  - head_lr = 1e-3
  - backbone_lr = 1e-5
  - batch_size = 8
  - epochs = 30
  - patience = 8

五、判定标准

如果相对当前 DINO 全学 frozen concat fold1：
- Accuracy / BAcc / F1 任意两项明显更高，视为正向
- AUC 不低太多，且高危召回不差，视为可扩 5-fold
- 否则不继续同配置，转两阶段路线
