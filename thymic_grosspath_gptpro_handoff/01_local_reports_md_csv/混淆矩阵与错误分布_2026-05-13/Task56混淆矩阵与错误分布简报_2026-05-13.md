# Task5/Task6 混淆矩阵与错误分布简报

日期：2026-05-13

这份简报只回答一个问题：目前几个关键模型最常把哪些类混在一起，错误更偏向哪一侧。

## 1. 当前给医生最值得看的三组结果

| 任务 | 模型 | Macro-F1 | Macro-AUC | 说明 |
|---|---|---:|---:|---|
| Task5 | DINOv2 vitl14 + whole | 0.6614 | 0.7959 | 三分类当前主结果 |
| Task6 | vits14 + vitb14 特征拼接 + Task5 层级先验 | 0.5127 | 0.8095 | 第一版正式最好结果 |
| Task6 | 结构化经验标签辅助版 | 0.5151 | 0.7963 | 后续经验标签增强版 |

说明：Task6 当前分数最高的版本是后续 cross-fit 融合版（F1-best 0.5232，AUC-best 0.8229）。本地保留了能直接复原矩阵的增强版是“结构化经验标签辅助版”，它最适合拿来说明经验标签把错误偏好往哪里推。

## 2. Task5 三分类：当前主结果

- 模型：DINOv2 vitl14 + whole
- Macro-F1 = 0.6614
- Macro-AUC = 0.7959
- 主要错误：B1-3 → A/AB：12 例；A/AB → B1-3：9 例；TC → B1-3：7 例；B1-3 → TC：5 例

这组结果说明，Task5 已经不是整体都难，而是错误主要集中在 TC 与 B1-3 的边界。A/AB 这一端相对更稳，真正拖后腿的是 TC 容易被判成 B 组。

## 3. Task6 六分类：第一版正式最好结果

- 模型：vits14 + vitb14 特征拼接 + Task5 层级先验
- Macro-F1 = 0.5127
- Macro-AUC = 0.8095
- 主要错误：B3 → B2：5 例；A → AB：4 例；B2 → B3：4 例；B2 → TC：4 例；TC → B2：4 例；A → B1：3 例

这版最清楚地暴露出三条难边界：A 与 AB、B1 与 B2、B2 与 B3/TC。其中 B2 是最典型的“边界中心类”，会同时向 B1、B3 和 TC 三侧扩散。

## 4. Task6 六分类：结构化经验标签辅助版

- 模型：结构化经验标签辅助版
- Macro-F1 = 0.5151
- Macro-AUC = 0.7963
- 主要错误：B2 → AB：5 例；A → B1：4 例；B2 → B3：4 例；A → AB：3 例；A → B3：3 例；B1 → A：3 例

这版的意义不是分数最高，而是能看出经验标签对错误偏好的影响。相较于第一版正式最好结果，它把一部分 TC/B2 方向的大错压下来了，但新的错误更多转向 A/AB 和 B2→AB 这一侧。也就是说，经验标签在帮助模型“学会看图理由”，同时也把模型的注意力重新分配到了另一组边界。

## 5. 医生最值得关注的边界

结合三张混淆矩阵，目前最值得医生重点回看的仍然是：

1. A 与 AB：哪些图从肉眼上本来就很难分。
2. B1 与 B2：哪些病例确实缺少清楚的切面或关键形态。
3. B2 与 B3：这条连续谱本来就是最难边界之一。
4. B2 / B3 与 TC：尤其是那些没有明显坏死、但病理是真正 TC 的病例。

## 6. 当前可以怎么理解这些图

- 如果一张图总是被多个模型稳定错到同一侧，通常说明它本身就处在视觉边界上。
- 如果经验标签辅助后，错误从一条边界转移到另一条边界，说明模型确实在学“经验规则”，而不是随机波动。
- 现阶段最需要的不是再给大量容易病例，而是补更多边界病例，尤其是 B2、TC 和 A/AB 边界样本。
