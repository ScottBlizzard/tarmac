# 项目阶段性工作详报

## 一、汇总说明

本报告基于项目启动至当前阶段形成的各类阶段报告、实验记录、医生版汇报、CSV 结果表、PDF/DOCX 导出材料以及已整理的“各阶段整理结果”撰写。重点说明项目各阶段实际开展了哪些工作、尝试了哪些方案、各项尝试结果如何、阶段内采用了什么方案、为什么这样选择、形成了哪些阶段产出，以及仍遗留哪些问题。

## 四、各阶段详细汇总

### 阶段1：基础复现与大体图可行性验证阶段

#### 1. 阶段定位

本阶段是项目的基础验证阶段，主要解决“胸腺大体图像是否值得建模”的问题。该阶段不追求最终模型，而是通过 baseline、SuRImage 复现和粗/细分类任务测试，判断大体照片中是否存在可学习的视觉信号，并建立后续实验的评估口径。

#### 2. 阶段进入原因

项目早期需要先确认两点：第一，常规胸腺大体照片中是否包含与病理风险或类别相关的图像信息；第二，已有的肺癌 SuRImage 等大体图像模型思路能否迁移到胸腺任务。报告中也明确区分了项目自建 `thymic_baseline` 与肺癌 SuRImage 原模型，说明本阶段的任务之一是复现和界定已有路线，而不是直接进入最终模型开发。

#### 3. 本阶段主要工作

本阶段主要开展了以下工作：

- 对早期胸腺大体图数据进行整理，形成可用于训练和验证的初步数据集。
- 复现 SuRImage 原始结构，包括 CoarseNet、FineNet、Fusion、CAM-guided crop 等模块。
- 建立 5-fold patient-level cross-validation，即按病例划分交叉验证，避免同一病例图像同时出现在训练和验证中。
- 使用病例级平均概率 `case-level mean probability` 汇总图像级预测。
- 测试低危/高危粗分层和 WHO 五分类等任务。
- 整理早期模型结果、错例分析和医生沟通材料，为后续任务定义和模型路线选择提供依据。

#### 4. 本阶段尝试与结果

| 任务 | 尝试过的模型/策略 | 最佳/阶段采用模型或策略 | 最佳结果数据 |
| -- | --------- | ------------ | ------ |
| 低危/高危粗分层 | `SE-ResNeXt50 + whole`、`SE-ResNeXt50 + whole+crop`、`SuRImage Stage1 coarse`、`SuRImage Fusion coarse` | `SuRImage Stage1 coarse` | AUC 0.7571，Accuracy 0.7100，BAcc 0.6875，F1 0.6133；作为“大体图存在粗分层信号”的阶段证据。 |
| WHO 五分类 | `SuRImage Stage1 fine`、`SuRImage Fusion fine`、`CAM-guided crop` 辅助细分类 | `SuRImage Stage1 fine`，但不作为后续主线 | Accuracy 0.3500，BAcc 0.3500，Macro-F1 0.3515，Macro-AUC 0.6538；细分类能力有限。 |
| Fusion 复现验证 | `SuRImage Fusion coarse`、`SuRImage Fusion fine` | 无正式采用策略，作为复现边界验证 | Fusion coarse AUC 0.6329；Fusion fine Macro-F1 0.2396，Macro-AUC 0.5734；未优于 Stage1。 |
| 旧主线优化策略 | `merged crop`、checkpoint 选择、阈值分析、label smoothing、类别不平衡处理、view routing、coarse/fine 分别选优 | 无正式采用策略，作为早期优化边界 | 旧主线五分类 Macro-F1 约 0.4523-0.456，Macro-AUC 约 0.705-0.720；边际收益有限，未作为后续主线。 |

#### 5. 阶段采用方案

本阶段采用的阶段方案是：保留 SuRImage strict reproduction 作为早期复现和对照证据；将 Stage1 coarse 的低危/高危结果作为“大体图中存在可学习风险分层信号”的阶段性证据；不将 SuRImage Fusion 或 WHO 五分类结果作为后续主结果。

#### 6. 选择原因

该选择主要基于结果稳定性和任务适配性。SuRImage 的粗分层结果显示大体图中有可学习信号，但 Fusion 和 WHO 五分类表现不稳定，说明直接迁移肺癌大体图模型不能解决胸腺细分类问题。考虑到当时每类样本有限，复杂融合结构容易放大小样本不稳定性，因此阶段内选择保留其可行性证据，而不是继续沿 SuRImage 作为主线。

#### 7. 阶段产出

本阶段产出包括：

- SuRImage strict reproduction 结果。
- 早期 baseline 结果。
- 5-fold patient-level cross-validation 评估口径。
- 病例级概率汇总方式。
- 早期错例分析和医生沟通材料。
- 对粗分层可行、细分类困难的阶段判断。

#### 8. 阶段结论

已验证结论：

- 胸腺大体图像中存在一定可学习信号。
- 低危/高危粗分层比 WHO 五分类更容易形成稳定结果。
- SuRImage 可作为早期复现参照，但其 Fusion 不适合直接作为本项目主力结果。

有信号但仍需验证的结论：

- 大体图中的具体视觉线索可能与风险分层有关，但本阶段尚未明确哪些线索最关键。

已否定或暂不采用的方向：

- SuRImage Fusion 作为主结果暂不采用。
- 直接依赖 CAM 裁剪进行胸腺 WHO 细分类暂不采用。

#### 9. 遗留问题

- WHO 五分类仍较难。
- 数据量小，单折样本有限。
- 哪些大体图形态真正对应低危/高危仍需进一步分析。
- 医生复核和正式标注体系尚未建立。

#### 10. 后续注意

- SuRImage Fusion、CAM-guided crop 细分类已经在本阶段显示不稳定，后续如无新增数据或明确机制改造，不建议重复投入大量时间。
- Stage1 coarse 的 AUC 只能作为“大体图存在粗分层信号”的阶段性证据，不能解释为胸腺 WHO 细分类已经可用。
- 本阶段指标来自早期数据和复现口径，不能与后续 Task7、selective diagnosis 或 clean cohort 结果直接横向比较。
- 如果后续重新使用 SuRImage，需要明确它是复现对照还是新方法模块，避免与后续 DINOv2/GrossPath-RC 主线混淆。

### 阶段2：Task1-Task6 任务体系与 DINO 强表征验证阶段

#### 1. 阶段定位

本阶段是任务体系建立和强表征验证阶段。其核心作用是把早期“能不能做”的问题推进到“哪些任务更适合做、哪些模型更有效”。该阶段建立了 Task1-Task6 的任务定义，并系统验证 DINOv2 等强表征模型相对旧 CNN/SuRImage 路线的提升。

#### 2. 阶段进入原因

基础复现阶段显示大体图有粗分层信号，但传统 CNN 和 SuRImage 在细分类上不稳定。因此需要重新梳理临床任务，并使用更强的视觉基础模型检验模型上限。医生也提出了更符合临床表达的任务，例如三分类和六分类，因此本阶段需要把任务体系和模型路线一起梳理清楚。

#### 3. 本阶段主要工作

- 整理 160 个病例、172 张图像的早期正式底座。
- 建立 Task1 至 Task6：
  - Task1：良性增生 vs 胸腺上皮肿瘤。
  - Task2：胸腺瘤 vs 胸腺癌。
  - Task3：低危 vs 高危。
  - Task4：WHO 五分类。
  - Task5：A/AB vs B1-3 vs TC。
  - Task6：A / AB / B1 / B2 / B3 / TC。
- 统一 5-fold 病例级交叉验证、病例级概率汇总、多分类 Macro-F1/Macro-AUC 等评估方式。
- 测试 DINOv2 vits14、vitb14、vitl14 等强表征模型。
- 尝试 DINO 与 PLIP 等模型融合。
- 建立医生复核材料、图像质量标签、经验标签核心集等辅助材料。

#### 4. 本阶段尝试与结果

| 任务 | 尝试过的模型/策略 | 最佳/阶段采用模型或策略 | 最佳结果数据 |
| -- | --------- | ------------ | ------ |

- Task6 六分类虽难，但并非完全不可做。

- 经验标签和层级先验对细分类有帮助，但稳定性仍需更多验证。

- SuRImage Fusion、ordinal loss、复杂 stacking、轻量微调暂不作为主方案。

#### 9. 遗留问题

- Task6 仍受小样本和边界病例限制。
- 经验标签直接接入训练不够稳定。
- 医生肉眼所见与模型特征之间尚未形成稳定、可审计的概念体系。

#### 10. 后续注意

- DINOv2 强表征是本阶段主要有效方向，后续若继续比较 backbone，应以同一 split、同一病例级汇总和同一指标口径进行，避免与本阶段结果不可比。
- Task5 和 Task6 的结果分别服务不同问题：Task5 更接近临床友好三分类，Task6 更偏细粒度研究型任务，不能把二者指标直接互相替代。
- 经验标签和 Task5 gate 有信号，但尚不能写成成熟、稳定的训练模块；后续使用时需要保留消融和失败结果。
- Task6 六分类受样本量限制明显，后续若继续推进，应优先补充边界亚型和医生确认标签，而不是只增加融合复杂度。

### 阶段3：Task7 新数据二分类冲刺与模型修补阶段

#### 1. 阶段定位

本阶段围绕 Task7 低危/高危二分类展开，是从多任务探索转向临床更关心的风险分层任务的模型冲刺阶段。其重点不是提出最终系统，而是在新合并数据上系统排查模型性能瓶颈，并确认哪些模型修补方向有效、哪些方向应停止。

#### 2. 阶段进入原因

医生更关注低危与高危风险分层，而新合并数据形成了 285 例 Task7 队列：低危 A/AB/B1 共 144 例，高危 B2/B3/TC 共 141 例。报告指出，二分类总量接近平衡，真正困难在于高危内部结构更复杂，尤其 B2 和 TC 增多、B3 偏少、边界病例更多。因此需要针对 Task7 做密集修补。

#### 3. 本阶段主要工作

- 建立 Task7 主基线：`DINOv2 vits14 + vitb14 concat`。
- 分析新旧数据六分类构成差异。
- 测试多图病例图1/图2/双图聚合策略。
- 测试视图感知策略，包括 cut、outer、mixed 分治。
- 测试 cut-only 和 full-to-cut refine。
- 测试普通 backbone 替代 DINO。
- 测试 DINO 微调和 Task6+Task7 联合监督。
- 优化 case-level/image-level MLP probe、seed 和 checkpoint 选择。

本阶段采用 DINO frozen strong feature + 稳定 probe/严格 fold-wise 融合方案。阶段内不采用 cut-only specialist、普通 backbone 替换、简单 DINO 微调或人为亚型加权作为主方案。

#### 6. 选择原因

本阶段最有效的提升来自同一套双 DINO 强特征内部的 probe 优化和严格融合，而不是换模型或拆视图。cut-only 虽然符合直觉，也有 AUC 信号，但 Accuracy/BAcc/F1 未超过统一基线，说明 cut 重要但不能孤立使用。普通 backbone 和可训练 DINO 的完整 5-fold 稳定性不足，因此不作为阶段主线。

#### 7. 阶段产出

- Task7 新数据主基线。
- 多图、视图、cut-only、backbone、微调等系统测试结果。
- 当前阶段最好严格融合结果。
- 多个负结果和不采用原因。

#### 8. 阶段结论

已验证结论：

- Task7 新数据比早期任务更难。
- DINO frozen feature 仍是最稳基础。
- probe 和 checkpoint 选择对结果有实际影响。

有信号但仍需验证的结论：

- cut 图像有重要信息，但单独 cut-only 不足以替代全局模型。
- 视图类型可能有帮助，但当前标签和样本规模不足。

已否定或暂不采用的方向：

- 人为亚型加权、普通 backbone 替代、简单 DINO 微调暂不采用。

#### 9. 遗留问题

- 阶段最好结果仍未达到临床期待的 90%。
- 低危误升和高危漏判机制尚需进一步拆解。
- 核心 hard 病例仍未解决。

#### 10. 后续注意

- cut-only、普通 backbone 替代、简单 DINO 微调和人为亚型加权均已做过系统尝试，当前不建议在相同数据和相同 recipe 下重复投入。
- `DINOv2 frozen feature + probe/fold-wise 融合` 是本阶段相对稳定方案，但仍只是 Task7 新数据阶段结果，不能解释为临床可部署结果。
- 多图、视图、cut/mixed/outer 的结论依赖当前视图标签和样本规模，后续若重新使用视图策略，应先补充更可靠的视图标注。
- 本阶段的 Accuracy/BAcc/F1 与后续课程学习、外部适配和 selective diagnosis 口径不同，交接时不要混作同一结果序列。

### 阶段4：Task7 难例分层、课程学习与经验标签阶段

#### 1. 阶段定位

本阶段用于解释 Task7 指标上限受限的原因。它不是单纯模型提分阶段，而是通过难例分层、课程学习、图像回看和经验标签尝试，判断模型到底是整体能力不足，还是少数系统性困难病例拖低整体结果。

#### 2. 阶段进入原因

Task7 修补阶段后，整体准确率仍在 0.7 左右。仅看总体指标无法判断问题来源。报告指出，需要先把病例分为 easy、medium、可救 hard 和核心 hard，确认哪些病例已经学稳，哪些病例属于系统性误导。

#### 3. 本阶段主要工作

- 使用多路模型表现对 285 例 Task7 病例做难度分层。
- 设计 Stage1 easy、Stage2 easy+medium、Stage3 加入可救 hard 的课程学习。
- 对 salvage hard 和核心 hard 进行图像回看。
- 总结低危误升和高危漏判的视觉误导线索。
- 构建 43 例强经验标签、204 例 weak+strong 标签、31 例高可信核心标签。
- 测试核心 hard 少量加入训练的影响。
- 整理医生复核清单。

#### 4. 本阶段尝试与结果

#### 5. 阶段采用方案

本阶段采用课程学习融合和难例分层作为阶段主方案。经验标签采取“少而精、用于解释和候选辅助”的方式，不把大规模弱标签或核心 hard 直接加权训练作为主方案。

#### 6. 选择原因

课程学习明确显示，模型在 easy/medium 和非核心 hard 上已经具有较高判别能力，主要瓶颈集中在核心 hard。相比盲目换模型或强行补标签，难例分层能更清楚地解释问题来源。大规模弱标签噪声较大，核心 hard 直接加入训练会扰动已学稳定边界，因此不作为阶段主方案。

#### 7. 阶段产出

- Task7 难度分层结果。
- Stage2+Stage3 课程学习融合结果。
- 核心 hard 医生复核清单。
- salvage_failed 和 regressed 病例图像回看。
- 经验标签多轮尝试结果。
- 错误机制总结。

#### 8. 阶段结论

已验证结论：

- 模型不是所有病例都差，easy+medium 和非核心 hard 表现较好。
- 核心 hard 是当前主要瓶颈。
- 课程学习能带来阶段性提升。

有信号但仍需验证的结论：

- 少量高可信经验标签可能有帮助，但稳定性不足。

已否定或暂不采用的方向：

- 大规模 weak 标签直接训练。
- 核心 hard 简单加权加入训练。

#### 9. 遗留问题

- 核心 hard 中哪些病例本身大体图不可分，需要医生确认。
- 经验标签如何转化为稳定模型模块仍未解决。
- 需要进一步区分可自动纠正病例和应复核/拒识病例。

#### 10. 后续注意

- 核心 hard 不是普通难例集合，已经验证过简单加权或少量 clean3 加入训练会扰动边界，后续不建议直接堆回主训练集。
- 31 例高可信经验标签只能作为有信号但不稳定的阶段结果，不能写成稳定提分模块。
- easy/medium、可救 hard、核心 hard 是基于当时模型体系的难度分层，不是病理学金标准；后续模型变化后需要重新审计分层有效性。
- 与 hard case 相关的病例清单应优先交给医生复核，特别是高危漏判和大体图本身可能不可分的病例。

### 阶段5：二阶段自动复核系统设计与早期验证阶段

#### 1. 阶段定位

本阶段从单阶段分类模型转向二阶段自动判读系统设计。阶段核心不是继续训练一个普通分类器，而是设计“可靠病例直接通过、疑难病例进入自动复核或纠偏”的流程。

#### 2. 阶段进入原因

课程学习后，`balcore blend` / 31 例核心经验标签候选结果达到 Accuracy 0.7684、BAcc 0.7685，但报告同时指出该结果稳定性弱于 Stage2+Stage3 正式主结果，仍未达到理想水平。继续单纯微调模型很难直接突破，因此本阶段需要先判断哪些病例可以放心自动判读，哪些病例应进入第二阶段复核。

#### 3. 本阶段主要工作

- 评估 `balcore blend` / 31 例核心经验标签候选结果的混淆矩阵和高危敏感性。该结果是候选融合/预测结果，不是独立 backbone 模型。
- 分析该候选结果的置信度、多模型分歧、方向性风险等信号。
- 设计一次通过门控器。
- 设计自动复核纠偏器。
- 提出保留/翻转模型、源模型选择器、高危漏诊专项纠偏器。
- 制定阶段 A/B/C 判断标准。
- 明确所有门控阈值必须采用嵌套 5-fold 方式选择，避免泄漏。

#### 4. 本阶段尝试与结果

| 任务 | 尝试过的模型/策略 | 最佳/阶段采用模型或策略 | 最佳结果数据 |
| -- | --------- | ------------ | ------ |
| Task7 候选预测结果评估 | `balcore blend / 31 例核心经验标签候选结果`、置信度与分歧分析 | 无正式采用策略，作为二阶段设计起点 | Accuracy 0.7684，BAcc 0.7685，高危敏感性 0.7730，TN/FP/FN/TP=110/34/32/109；稳定性弱于 Stage2+Stage3。 |

#### 5. 阶段采用方案

本阶段采用“先提升一次通过池，再训练自动复核纠偏器”的阶段方案。报告明确不将“人工复核可纠正”包装成“自动纠偏已经成功”。

#### 6. 选择原因

如果一次通过比例太低，系统会把大量病例推给复核器，复核器压力过大且容易误伤原本正确病例。当前高置信度直接通过池仅约三分之一病例，说明最紧急的问题是可靠性识别，而不是立即训练一个强制纠偏器。

#### 7. 阶段产出

- 二阶段自动判读系统执行计划。
- 一次通过门控器指标体系。
- 自动复核纠偏器设计。
- hard_core 拆分方向。
- 阶段 A/B/C 判断标准。
- Gradient Boosting / GB 自动复核候选结果。

#### 8. 阶段结论

已验证结论：

- 当前单模型还不足以达到理想自动诊断水平。
- 模型置信度和分歧信号能识别部分需复核病例。

有信号但仍需验证的结论：

- 自动复核纠偏器可能有效，但尚未形成稳定端到端结果。

已否定或暂不采用的方向：

- 不能把人工假设纠正结果写成纯自动模型结果。

#### 9. 遗留问题

- 一次通过池比例仍偏低。
- 自动复核器尚未完成稳定验证。
- hard_core 哪些可自动纠正、哪些必须复核仍未确定。

#### 10. 后续注意

- 本阶段主要是系统设计和早期信号验证，不能把“假设复核可纠正”写成“自动复核器已经纠正”。
- 一次通过门控器仍是后续接手重点；如果一次通过池过小，直接训练纠偏器的系统价值有限。
- 保留/翻转、源模型选择器、高危漏诊纠偏器均需严格 OOF 或冻结验证，不能在全量 OOF 上直接选阈值。
- hard_core 的自动纠正和医生复核边界需要继续拆分，尤其要单独追踪 FN，即高危被判低危的病例。

### 阶段6：第三批数据适配与严格外部压力测试阶段

#### 1. 阶段定位

本阶段用于检验模型面对新增批次和严格外部数据时的泛化边界。它的重要作用是明确：内部表现好并不等于外部表现稳定，第三批数据和严格外部数据必须按照不同角色解释。

#### 2. 阶段进入原因

第三批 306 例数据引入后，旧模型直接外推明显下降。第三批虽然来自相近体系，但类别构成、图像亮度、主体大小、背景比例、拍摄构图等都与旧数据不同。同时严格外部集表现更低，因此需要系统进行适配和压力测试。

#### 3. 本阶段主要工作

- 定义旧数据、第三批、严格外部集的角色。
- 分析第三批类别构成：低危 224 例，高危 82 例，AB 占比高，A 和 B3 缺失。
- 将第三批拆分为 adapt72 和 holdout234。
- 比较旧模型直接外推、第三批适配版（旧数据 + adapt72 保守校准）、参考上限版。
- 测试 DINOv3、crop/whole 微调、head-only 微调、融合等补充方向。
- 对严格外部集做冻结评估和质量门控分析。

#### 6. 选择原因

第三批已经参与观察和适配，如果继续称为严格外部测试，会造成证据边界不清。第三批适配版（旧数据 + adapt72 保守校准）没有使用 holdout 标签调参，可信度高于参考上限，但仍属于第三批适配口径结果。严格外部表现较低，说明现阶段不能宣称强跨域部署能力。医学上还必须重视高危漏判，不能只追求总体 Accuracy。

#### 7. 阶段产出

- 第三批数据角色定义。
- 第三批适配/holdout 结果。
- 严格外部冻结评估结果。
- 质量门控和 GPT 质量标签记录。
- 对外部域偏移的阶段判断。

#### 8. 阶段结论

已验证结论：

- 第三批直接外推下降，说明同体系新批次仍存在分布偏移。
- 小样本适配可提升第三批表现。
- 严格外部强制分类不足，特别是高危召回较低。

有信号但仍需验证的结论：

- DINOv3 可作为高危敏感分支或分歧信号。
- 质量门控对外部有帮助，但不是根本解决方案。

已否定或暂不采用的方向：

- 将第三批写成严格外部。
- 使用 holdout 标签筛选出的 0.87 结果作为正式可部署结果。
- 单纯微调 DINO 作为主线。

#### 9. 遗留问题

- 严格外部泛化未解决。
- 高危漏判仍是核心风险。
- 需要更多未暴露外部数据。
- 外部错误来源仍需结构化分析。

#### 10. 后续注意

- 第三批数据已经参与观察、适配和策略选择，后续文档中不能再称为 strict external 或完全干净外部测试集。
- 使用 holdout 标签筛选得到的 0.87 参考上限只能用于判断潜在空间，不能作为可部署模型结果。
- 严格外部强制分类下降是重要边界，不应被后续 workflow 的选择性结果覆盖或淡化。
- DINOv3、质量门控和 crop 微调均有局部信号，但尚未解决外部泛化，后续应围绕错误来源、质量/视角/主体尺度分层补实验。

### 阶段7：项目证据边界与 GrossPath-RC 框架重构阶段

#### 1. 阶段定位

本阶段不是单纯写论文，而是对项目证据边界和表达方式进行重新定位。其核心作用是避免把阶段性模型结果包装成成熟外部泛化产品，同时将现有工作整理成更符合医学和计算机审稿逻辑的风险可控框架。

#### 2. 阶段进入原因

严格外部集结果不支持“强外部泛化分类器”的叙述。第三批也已参与适配，不能作为干净外部集。报告因此提出，项目不应继续只围绕单个分类器分数写作，而应聚焦信号源、任务定义、风险控制、外部域偏移和临床复核流程。

#### 3. 本阶段主要工作

- 固定数据角色：旧数据为开发队列，第三批为扩展开发/适配队列，严格外部为压力测试或冻结验证。
- 梳理论文可写和不可写的主张。
- 对照 SuRImage、临床照片 AI、foundation model、concept bottleneck、domain generalization、selective prediction 等相关范式。
- 提出 GrossPath-RC 框架命名。
- 将项目从“低危/高危分类器开发”重构为“概念引导、域感知、风险可控的大体病理图像风险分层框架”。

#### 4. 本阶段尝试与结果

| 任务 | 尝试过的模型/策略 | 最佳/阶段采用模型或策略 | 最佳结果数据 |
| -- | --------- | ------------ | ------ |
| 项目证据边界重构 | 强外部泛化分类器写法、第三批干净外部写法、风险可控工作流写法 | 风险可控工作流写法 / GrossPath-RC 框架 | 无新增独立性能指标；依据前序 strict external 强制分类 Acc 0.6381、BAcc 0.6220 判断不能写成强外部泛化分类器。 |
| Task7 主任务定位 | Task6 六分类主线、Task7 低危/高危风险分层主线 | Task7 低危/高危风险分层作为主任务 | 无新增独立模型指标；基于旧数据、第三批适配和 strict external 边界综合确定。 |
| 概念引导框架 | 医生概念、concept bottleneck、concept heads、BiomedCLIP/PLIP/CONCH 概念对齐候选 | 保留为 GrossPath-RC 框架模块 | 无新增性能指标；自动概念模块需后续验证。 |
| 选择性诊断 / risk-controlled workflow | selective prediction、conformal risk control、质量/视图/域/分歧信号 | 作为后续主表达方向 | 无新增性能指标；后续以自动覆盖率、复核/拒识比例、自动错误数和 Wilson95 上界评估。 |

#### 5. 阶段采用方案

阶段采用 GrossPath-RC 作为后续方法框架的表达方向：Concept-guided、Domain-aware、Risk-controlled。Task7 低危/高危为临床主任务；质量、视角、域、概念、模型分歧和拒判/复核纳入统一工作流。

#### 6. 选择原因

该方向更符合当前证据边界。项目已经证明大体图有可学习信号，但严格外部强制分类不足；如果继续写成全自动分类器，会与数据表现不一致。风险控制工作流能够真实表达项目优势，同时保留外部失败和复核需求。

#### 7. 阶段产出

- 论文主线与保守贡献路线报告。
- Nature 子刊方向重构材料。
- GPTPro 转换报告。
- GrossPath-RC 框架草案。
- 可写/不可写 claim 清单。

#### 8. 阶段结论

已验证结论：

- 不能把项目写成成熟全自动外部泛化分类器。
- Task7 风险分层比 Task6 六分类更适合第一主线。
- 数据边界必须严格固定。

有信号但仍需验证的结论：

- 概念引导和选择性诊断可成为方法贡献，但仍需更多实验支持。

已否定或暂不采用的方向：

- 宣称替代病理诊断。
- 宣称第三批为严格外部。
- 宣称外部探索后最高分为干净外部验证。

#### 9. 遗留问题

- 缺少新的未暴露外部验证。
- 缺少医生盲读和 AI 辅助实验。
- 概念标签尚未形成正式可审计体系。
- 算法消融仍需进一步整理。

#### 10. 后续注意

- 本阶段是证据边界和写法重构，不是实验性能突破阶段；后续引用时不要把框架建议写成已完成的最终系统。
- GrossPath-RC 是当前推荐的组织框架，但其中概念引导、域感知、risk-control 各模块仍需要分别用实验支撑。
- 严格外部、第三批、旧数据的角色必须保持一致，不能为了写作便利混用或重新命名。
- 若准备对外材料，应保留“不能宣称替代病理诊断、不能宣称全自动成熟部署、不能把已暴露外部写成干净验证”的边界。

#### 5. 阶段采用方案

本阶段保留多模型一致性和质量门控作为当前较可靠的风险控制信号；医生概念作为解释和监督层保留；不采用 v1 stacking 或域内 guard 作为主结果；使用 v2 清单指导后续域泛化训练。

#### 6. 选择原因

多模型一致性在外部数据上比域内 guard 更稳。stacking 没有超过基础模型，说明简单堆模型不能解决外部泛化。医生概念有信息价值，但当前图像到概念的预测能力不足，直接拼接无法稳定提分。

- GrossPath-RC v0 完整 workflow 结果。
- GrossPath-RC v1 冻结式 workflow 否证结果。
- GrossPath-RC v2 域泛化诊断报告。
- 三类训练清单：
  - safe core：214 例。
  - hard error：74 例。
  - domain focus：384 例。

已验证结论：

- 外部全量强制分类不足。
- 多模型一致性和质量门控可筛出更可靠子集。
- stacking 和域内 guard 不适合作为当前主方案。

有信号但仍需验证的结论：

- 医生概念和 image-to-concept 可作为后续概念监督方向。
- 域泛化训练可围绕质量、视角、主体尺度和概念展开。

已否定或暂不采用的方向：

- 直接使用 stacking 作为主分类器。
- 将域内 guard 直接部署到外部。

#### 9. 遗留问题

- 外部自动覆盖率有限。
- 概念预测头尚弱。
- 视角标签覆盖不足，不能支撑全量视角结论。
- 多目标训练尚未完成。

#### 10. 后续注意

- v1 stacking 和 reliability guard 已经在冻结外部评估中显示不足，后续不建议在相同输入上反复调参包装为主结果。
- 多模型一致性和质量门控是当前更稳的风险控制信号，但它们会降低自动覆盖率，交接时必须同时报告 coverage 和 review/reject。
- image-to-concept 有探索价值，但概念预测 AUC 仍偏弱，不能作为成熟概念瓶颈模型使用。
- v2 训练清单是后续域泛化训练入口，使用时应区分 safe core、hard error、domain focus 的用途，不要混成单一训练样本池。

### 阶段9：安全释放评分卡、无标签偏移门控与 clean cohort 阶段

#### 1. 阶段定位

本阶段是在风险控制框架内进一步提高自动释放效率，并明确队列定义和安全边界。其重点不是证明全量自动诊断已经完成，而是在允许复核/拒识的工作流口径下，尽量减少自动放行错误，同时降低复核负担。

#### 2. 阶段进入原因

GrossPath-RC v0-v2 证明选择性诊断有价值，但外部覆盖率和安全边界仍需更严格定义。同时，医生确认了部分特殊组织学或混杂病例，需要建立 clean cohort 和 full cohort sensitivity 机制，避免后续报告被质疑为选择性剔除。

#### 3. 本阶段主要工作

- 构建 two-signal scorecard。
- 设计高安全评分卡、安全释放评分卡和固定稳定释放规则。
- 比较低置信度复核、方向感知复核、高安全兜底、安全释放。
- 建立无标签 severe-shift gate。
- 比较 v118、v161、v180、v182、v185、v195、v201。
- 建立医生确认的 clean cohort exclusion list。
- 计算自动错误率和 Wilson95 上界。

#### 5. 阶段采用方案

本阶段采用 selective diagnosis / risk-controlled workflow 口径进行解释；v195 作为当前阶段主流程；clean cohort 作为主分析，full cohort 作为敏感性分析；v201 仅作为候选增强模块，不作为成熟结论。

#### 6. 选择原因

v195 相比 v185 将自动判读数量从 287 例提升到 447 例，复核/拒识比例从约 58.94% 降到约 36.05%，同时自动错误仍为 1 例。v180 虽然效率更高，但释放错误增加，不符合安全优先。clean cohort 是基于医生确认的特殊组织学/任务定义问题，而不是按模型错误临时剔除，因此更符合内部复盘、后续交接和对外沟通时对数据治理的要求。

#### 7. 阶段产出

- 高安全评分卡。
- 安全释放评分卡。
- 无标签偏移门控。
- v195 workflow 结果。
- v37-v86 等历史风险控制 workflow 记录。
- clean/full cohort 结果。
- 医生确认剔除病例清单。
- 自动错误 Wilson95 上界。

#### 8. 阶段结论

已验证结论：

- 当前高分必须解释为允许复核/拒识的风险控制工作流结果。
- 自动释放能在保持较高安全性的同时降低复核负担。
- clean cohort 与 full cohort 结果接近，说明剔除特殊病例不是人为提分主要来源。

有信号但仍需验证的结论：

- v201 自动纠偏有正信号，但样本量太小。
- 无标签 severe-shift gate 有框架价值，但还需前瞻校准。

已否定或暂不采用的方向：

- 贪心释放规则不采用。
- 自动翻转/纠偏不能作为成熟主模块。
- 严格外部 100% 点估计不能解释为普适全自动诊断成功。

#### 9. 遗留问题

- 严格外部复核/拒识比例仍较高，约一半病例需要复核或拒识。
- 外部样本量小，置信区间仍宽。
- v201 自动纠偏仍需更多样本验证。
- 后续仍需前瞻验证、更多外部数据和医生工作流实验。

#### 10. 后续注意

- v195 的高指标是 selective diagnosis / risk-controlled workflow 口径，不是全量病例完全自动诊断口径，后续交接和对外沟通必须同时给出自动覆盖率和复核/拒识比例。
- 严格外部 100% 点估计不能单独使用；外部自动样本数和 Wilson95 上界必须一起报告。
- v201 自动纠偏目前只救回少量病例，仍是候选增强模块，不能写成成熟自动纠偏。
- clean cohort 与 full cohort 的口径要固定；医生确认 exclusion list 不应随模型结果临时变化。

## 五、指标与口径说明

| 术语 / 指标 | 含义 | 本项目中需要注意的口径 |
| ---- | ---- | ---- |
| AUC | Area Under the ROC Curve，衡量模型把阳性样本排在阴性样本前面的能力，偏重排序能力 | AUC 高不一定代表最终分类准确率高；在 Task7 中还需要同时看 BAcc、FN/FP 和高危召回 |
| Accuracy / Acc | 准确率，即预测正确病例数占总病例数比例 | 当低危/高危比例不均衡时，Accuracy 容易被多数类影响，不能单独作为医学安全指标 |
| BAcc | Balanced Accuracy，平衡准确率，通常为敏感性和特异性的平均 | Task7 低危/高危风险分层中更适合与 Accuracy 同时报告，尤其用于类别比例变化较大的第三批和外部集 |
| Macro-F1 | 多分类中分别计算每类 F1 后取平均，避免多数类主导结果 | 主要用于 Task4-Task6 等多分类任务；不能直接与 Task7 二分类 BAcc 横向比较 |
| Macro-AUC | 多分类任务的类别级 AUC 汇总 | 常用于 Task4-Task6，表示多分类排序能力；不等同于最终 argmax 分类效果 |
| Sensitivity / 高危敏感性 / 高危召回 | 在高危病例中模型正确识别为高危的比例 | 对 Task7 医学风险最关键；高危漏判对应 FN，需要单独报告 |
| Specificity / 低危特异性 | 在低危病例中模型正确识别为低危的比例 | 反映低危误升高危的控制能力；需与高危敏感性平衡 |
| FN | False Negative，在 Task7 中通常指高危被判为低危 | 医学风险较高，后续复核和风险控制时需优先关注 |
| FP | False Positive，在 Task7 中通常指低危被判为高危 | 会造成误升级或过度复核，但医学风险通常不同于 FN，需要分开分析 |
| OOF | Out-of-fold，交叉验证中每个病例只在其未参与训练的折上得到预测 | 用于内部开发和模型选择时较可靠，但仍不等同于外部验证 |
| 5-fold patient-level cross-validation | 5 折病例级交叉验证，按病例划分训练/验证，避免同一病例泄漏 | 本项目多阶段使用该口径；若图像级划分会导致泄漏，因此应坚持病例级 |
| fold-wise | 按每一折独立选择阈值、权重或策略，再应用到该折测试部分 | 可降低阈值选择泄漏风险；不能用全量 OOF 最优阈值冒充 fold-wise 结果 |
| frozen feature | 冻结预训练 backbone，只提取特征，不更新 backbone 参数 | DINOv2 frozen feature 是多个阶段的稳定基础；与可训练微调结果需分开比较 |
| probe | 在冻结特征上训练的轻量分类头，如 logistic regression、MLP 等 | 本项目中 probe 优化带来过有效提升，但需按同一 split 和 fold-wise 规则比较 |
| clean cohort | 根据医生确认的任务边界或特殊组织学原因形成的主分析队列 | 不是按模型错误临时删除；应与 full cohort sensitivity 同时报告 |
| full cohort | 包含全部可追溯病例的完整队列，用于敏感性分析 | 用于证明 clean cohort 处理不改变主结论，不能与 clean cohort 混写 |
| third batch / 第三批 | 后续新增的 306 例数据，在项目中已用于观察、适配和策略选择 | 不应再写成 strict external；建议称为扩展开发队列、时间外开发队列或适配队列 |
| strict external | 严格外部测试或外部压力测试数据，原则上不用于训练、调参和策略选择 | 本项目外部集样本量较小且部分结果为 workflow 口径，解释时必须谨慎 |
| holdout | 从某批数据中留出的评估部分 | 如果 holdout 标签被用于筛选策略，则相关结果只能作为参考上限，不能作为可部署正式结果 |
| selective diagnosis | 选择性诊断，即系统只对部分病例自动给出低危/高危，其余进入复核、拒识或重拍建议 | 后期高指标多属于该口径，不能解释为全量病例完全自动诊断 |
| 自动覆盖率 | 在选择性诊断中，被系统自动放行或自动判读的病例比例 | 必须和自动准确率、复核/拒识比例一起报告；覆盖率越高通常安全压力越大 |
| 复核/拒识比例 | 系统没有自动给出最终诊断，而是建议复核、拒识或人工确认的病例比例 | 高复核比例说明系统更保守，不能只看高 BAcc 而忽略复核负担 |
| 自动错误数 | 自动放行或自动判读子集中发生错误的病例数 | 是选择性诊断安全性的核心指标，应与自动覆盖率一起报告 |
| Wilson95 上界 | 用 Wilson score interval 估计错误率 95% 置信区间上界 | 当自动样本数少时，即使观察到 0 错误，上界也可能较宽；不能只报 0 错误 |
| severe-shift gate | 无标签或弱标签条件下判断当前批次是否发生严重域偏移的门控 | 当前属于风险审计和候选部署逻辑，仍需前瞻批次校准 |
| risk-controlled workflow | 风险控制工作流，强调自动放行、复核/拒识、质量门控、偏移门控等联合使用 | 不是单一分类器；报告时必须说明哪些病例自动处理，哪些病例被保留给复核 |
