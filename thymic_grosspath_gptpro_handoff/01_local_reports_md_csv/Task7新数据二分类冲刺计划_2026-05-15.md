# Task7 新数据二分类冲刺计划

日期：2026-05-15

## 1. 当前目标

接下来这一轮工作的主目标，不再是继续分散推进多个任务，而是集中解决一个问题：

- 在合并第一批和第二批数据后的新数据集上，把临床最关心的二分类任务做强。

这里的主任务定义为：

- 低危组：`A / AB / B1`
- 高危组：`B2 / B3 / TC`

也就是当前的 `Task7`。

我们希望最终能做到：

- 在新数据上，`Task7` 的总体准确率尽量冲到 `90%+`
- 同时尽量减少高危病例被误判回低危的情况

这里要特别说明：

- 单纯的 `Accuracy` 还不够
- 后续每一步都要同时看：
  - `Accuracy`
  - `Balanced Accuracy`
  - `AUC`
  - `Sensitivity（高危召回）`
  - `Specificity`
  - `高危误判为低危的例数`

原因很简单：新数据已经不再是均衡数据，如果只看 `Accuracy`，模型可能靠偏向多数类把数字抬高，但临床上最重要的“高危不要漏”反而会变差。

## 2. 当前基础判断

### 2.1 目前已经看到的现象

1. 在旧数据上，某些模型的 `Task6` 六分类结果折叠成高低危后，反而比直接训练 `Task7` 二分类更少漏掉高危。
2. 在新数据上，第二批数据加入后，`Task5` 还能小幅提升，但 `Task6` 和 `Task7` 都出现了不同程度倒退。
3. 新数据的类别构成已经明显不均衡，尤其 `AB`、`B2`、`TC` 较多，`B3` 明显偏少。
4. 新数据的多图病例目前统一取第二张图，这个选择本身可能影响结果。
5. 医生已经提出了三个非常值得纳入实验的方向：
   - 蓝色背景板做颜色标准化
   - 引入肿瘤大小信息
   - 分析模型注意力是否落在肿瘤区，而不是脂肪或背景
6. 医生还提出了一个很重要的临床判断：
   - `A` 型在临床中少见，但在当前细分类里可能是一个“陷阱亚型”
   - 它可能会明显拉低模型表现，却未必在真实世界里造成同等规模的临床风险

### 2.2 这轮冲刺的基本策略

这一轮不再大面积换模型，不再广撒网。

我们只做三件事：

1. 围绕 `Task7` 主任务，系统优化数据策略、监督方式和训练口径。
2. 让 `Task6 -> Task7` 折叠结果和直接 `Task7` 结果在同一套新数据上长期并行比较。
3. 把医生提出的临床想法，变成可以被验证的实验因素，而不是停留在讨论层面。

## 3. 当前最重要的实验主线

后续所有实验，统一围绕下面两条主线推进：

### 主线 A：直接二分类

- 直接训练 `Task7`
- 输出 `low_risk_group / high_risk_group`

这是最符合医生直觉、也最容易直接汇报的主线。

### 主线 B：先六分类，再折叠成二分类

- 先训练 `Task6`
- 再把 `A / AB / B1` 合成低危
- 把 `B2 / B3 / TC` 合成高危

这条线目前已经有迹象说明：

- 它有可能比直接二分类更少漏掉高危

所以后面所有关键改进，原则上都要同时看：

- 对直接 `Task7` 是否有帮助
- 对 `Task6 -> Task7` 是否更有帮助

## 4. 实验组织方式

这一轮实验不再按“想到一个就全量跑一个”的方式推进，而改成四层漏斗：

### 第 1 层：零成本重分析

先不训练，直接利用已有输出做验证，例如：

- `Task6` 结果重新折叠成 `Task7`
- 重新算混淆矩阵
- 重新统计高危漏诊数
- 分来源（第一批/第二批）对比

### 第 2 层：轻量 A/B

只改一个因素，在当前最强模型上做对照，例如：

- 是否做蓝板颜色标准化
- 是否加入尺寸信息
- 是否改用切面优先主图

### 第 3 层：单模型完整 5-fold

只有第 2 层出现明确正信号，才升级到完整 `5-fold patient-level CV`。

### 第 4 层：公平对照扩展

只有已经证明确实有效的策略，才扩到多个模型公平比较。

## 5. 实验清单总览

下面所有实验，按优先级分成 `P0 / P1 / P2 / P3`。

- `P0`：必须先做，信息量最大
- `P1`：强烈建议做，最可能直接带来提升
- `P2`：值得做，但要看前面结果
- `P3`：探索性，前面无明显提升再考虑

## 6. P0：先把现有结果吃透

### P0-1 统一重算所有新数据模型的 `Task6 -> Task7` 折叠结果

目的：

- 确认在新数据上，到底是直接二分类更强，还是六分类折叠更强
- 不再只看单个模型，统一比较所有强模型

具体做法：

- 对新数据上已跑完的 `Task6` 模型全部折叠成 `Task7`
- 输出：
  - `Accuracy`
  - `Balanced Accuracy`
  - `AUC`
  - `Sensitivity`
  - `Specificity`
  - `F1`
  - `高危误判为低危的例数`

成功标准：

- 找出新数据上最值得主推的“临床二分类路径”
- 明确是：
  - 直接 `Task7`
  - 还是 `Task6 -> Task7`

### P0-2 新旧数据来源分层分析

目的：

- 确认第二批数据到底是“增加难度”还是“改变分布”

具体做法：

- 对当前最好模型分别在：
  - `batch1`
  - `batch2`
  上单独统计结果
- 重点看：
  - 高危漏诊数
  - `TC`
  - `B2`
  - `B3`
  - `A / AB`

成功标准：

- 明确倒退主要来自：
  - 域差异
  - 类别不平衡
  - 主图选择
  - 还是某些类本身极难

### P0-3 多图病例策略回顾

目的：

- 判断“统一取第二张图”是不是一个错误决策

具体做法：

- 列出所有多图病例
- 比较：
  - 第二张图单独判断
  - 第一张图单独判断
  - 两张图聚合
- 重点分析医生已经问过的病例和高危误判病例

成功标准：

- 得出后续多图病例的默认策略：
  - 第二张优先
  - 切面优先
  - 双图聚合优先

## 7. P1：最值得直接尝试的训练改进

### P1-1 蓝色背景板颜色标准化 A/B

来源：

- 医生建议：以蓝色背景板为参考，统一色温和曝光

目的：

- 降低拍摄条件差异
- 提高第一批和第二批图像的一致性

具体做法：

- 以蓝色背景板区域为颜色参考
- 生成一版统一白平衡/色温/亮度的图像
- 只在当前最强 `Task7` 模型上做 A/B：
  - 原图
  - 标准化后图

先跑模型：

- `DINOv2 vitb14 + whole`
- `DINOv2 vits14+vitb14 concat`

成功标准：

- `Accuracy` 提升至少 `+0.02`
  或
- 高危漏诊数下降至少 `2-3` 例

若无改善：

- 这条线降级为补充，不扩到全家桶

### P1-2 多图病例恢复双图聚合

目的：

- 验证当前“每例只取一张图”是否损失重要信息

具体做法：

- 对多图病例恢复：
  - 图1
  - 图2
  - 双图概率聚合
- 单图病例保持原样

优先任务：

- `Task7`
- `Task6 -> Task7`

成功标准：

- 高危漏诊数下降
- `Sensitivity` 提升
- 不显著牺牲 `Specificity`

### P1-3 切面优先主图策略

来源：

- 医生连续多次追问某些病例用的是哪张图，尤其是否为切面图

目的：

- 检验切面图是否比当前默认第二张图更有诊断价值

具体做法：

- 多图病例重新指定主图：
  - 优先切面图
  - 若两张均为切面或均非切面，再按现有规则
- 与当前第二张策略做 A/B

成功标准：

- 高危漏诊数下降
- `Task7` 二分类错误方向更符合临床预期

### P1-4 类别不平衡正式入模

目的：

- 当前新数据已经不均衡，训练口径必须跟上

具体做法：

- 对 `Task7` 和 `Task6` 分别试：
  - weighted cross-entropy
  - class-balanced sampler
  - focal loss（仅在前两者无效时试）

重点观察：

- 高危召回
- 高危误判为低危的例数

成功标准：

- 高危召回明显上升
- 总体 `Balanced Accuracy` 不下降

### P1-5 重新校准阈值，而不是死守 0.5

目的：

- 二分类里，固定 `0.5` 不一定是最临床合适的决策点

具体做法：

- 在验证集上选阈值
- 主目标设为：
  - 高危召回优先
  - 同时控制低危误报不要过高

成功标准：

- 比固定阈值少漏高危
- 报告中同时给出：
  - 默认阈值结果
  - 临床优化阈值结果

## 8. P2：医生建议转成结构化增强信息

### P2-1 肿瘤大小作为结构化特征加入

来源：

- 医生建议：如果模型不懂尺子，可以直接提供中径/最大径

目的：

- 看尺寸信息是否能帮助二分类

具体做法：

- 不让模型直接“读尺子”
- 由医生或台账提供：
  - 最大径
  - 中径
  - 或最稳定的一个尺寸指标
- 做后端融合：
  - image features
  - size feature

优先模型：

- `DINOv2 vitb14 + whole`
- `DINOv2 vits14+vitb14 concat`

额外分析：

- 误判是否集中在小肿瘤
- 高危小病灶是否更容易漏

成功标准：

- `Task7` 整体指标提高
  或
- 高危小病灶漏诊明显减少

### P2-2 经验标签继续只做结构化，不再走自由文本

目的：

- 经验标签已经证明有信息量
- 但自由文本直接训练不稳

具体做法：

- 保留结构化经验标签辅助头
- 重点只保留最关键字段：
  - `boundary_axis`
  - `tc_impression`
  - `lowrisk / bgroup impression`
  - `conflicting_clues`

不再做：

- 原始自由文本直接监督

成功标准：

- 比无经验标签版本更少漏高危
- 或六分类折叠后更稳

### P2-3 医生人工修订经验标签只保留“机器能吃的部分”

目的：

- 人工修订版说明人类经验有价值
- 但自由表述太散，不适合直接训

具体做法：

- 把人工修订内容继续压缩成少量结构化字段
- 优先用于：
  - `A / AB`
  - `B2 / B3 / TC`
这些边界病例

成功标准：

- 局部边界改善
- 至少在错例分析上更可解释

## 9. P2：模型注意力与 ROI 路线

### P2-4 当前最强模型注意力热图

来源：

- 医生建议：看看模型到底盯的是肿瘤还是周围脂肪

目的：

- 判断 whole-image 的有效信息到底来自哪里

具体做法：

- 对当前最好 `Task7` 模型输出注意力/热图
- 重点看：
  - 高危漏诊病例
  - `TC`
  - `B2/B3`
  - `A` 陷阱病例

成功标准：

- 明确模型主要盯：
  - 肿瘤主体
  - 边缘
  - 脂肪
  - 背景板

### P2-5 ROI 限制版 vs 整图版

目的：

- 回答“只看肿瘤区会不会更好”

具体做法：

- 在有代表性的子集上做：
  - whole
  - ROI/crop
  - whole+ROI

前提：

- 先有注意力热图依据
- 如果当前热图显示确实被背景带偏，再推进这条线

成功标准：

- 如果 ROI 版明显减少背景误导，就升级为正式比较项

## 10. P3：任务定义层面的临床简化实验

### P3-1 `A` 并入 `AB` 的临床简化五分类

来源：

- 医生判断 `A` 可能是低频但高干扰的“陷阱亚型”

目的：

- 验证是否需要在临床主任务里降低 `A` 的独立地位

具体做法：

- 新建一个简化版任务：
  - `AB_plus_A`
  - `B1`
  - `B2`
  - `B3`
  - `TC`
- 看：
  - 跨等级错误是否下降
  - 高危漏诊是否减少

成功标准：

- 如果五分类明显更稳，后续可以把它作为临床补充任务

### P3-2 把 `Task7` 定成最终主任务、`Task6` 作为辅助训练任务

目的：

- 正式验证“细分类辅助风险分层”这条主假设

具体做法：

- 同一个 backbone
- 一个头输出 `Task6`
- 一个头输出 `Task7`
- 联合训练

成功标准：

- 比单独 `Task7` 更少漏高危
- 比单独 `Task6 -> Task7` 折叠更稳

这条线是最可能形成方法学亮点的新模型方向，但不放在最前面，因为前面还有更低成本、更高信息量的因素要先验证。

## 11. 每个实验都要统一输出什么

后面每个有效实验，统一输出以下内容：

1. 总指标表
   - `Accuracy`
   - `Balanced Accuracy`
   - `AUC`
   - `Sensitivity`
   - `Specificity`
   - `F1`

2. 混淆矩阵
   - 原始计数
   - 行归一化比例

3. 高危误判为低危病例清单

4. 分来源对比
   - `batch1`
   - `batch2`

5. 若涉及多图病例：
   - 主图策略说明
   - 是否切面优先
   - 是否双图聚合

6. 若涉及经验标签：
   - 用了哪些字段
   - 覆盖多少病例
   - 是否结构化

## 12. 本轮冲刺建议的实际执行顺序

建议严格按下面顺序推进，不要乱序：

### 第一阶段：先做不重训或轻量重训

1. 全部 `Task6` 折叠成 `Task7` 再比较
2. 新旧数据来源差异分析
3. 多图病例策略回顾
4. 蓝色背景板标准化 A/B
5. 阈值重校准

### 第二阶段：在当前最强模型上做针对性训练

6. 类别不平衡训练策略
7. 切面优先主图策略
8. 结构化经验标签辅助
9. 加尺寸特征

### 第三阶段：再考虑新任务与结构升级

10. `A` 并入 `AB` 的临床简化版
11. `Task6 + Task7` 联合训练
12. ROI/attention 驱动的区域建模

## 13. 停损原则

为了避免再次陷入“试了很多，但没有形成主线”的状态，这一轮设定明确停损条件：

- 任一新因素，如果在当前最强模型上：
  - `Accuracy` 没有提升
  - `Balanced Accuracy` 没有提升
  - 高危漏诊也没有减少
  那么这条线立即降级，不再扩到全家桶。

- 只有同时满足下面至少一条，才升级：
  - `Accuracy` 提升明显
  - 高危漏诊数减少
  - `AUC` 明显提升
  - 医生关心的错误方向明显改善

## 14. 这一轮最值得期待的突破点

从当前信息看，最可能真正带来明显提升的，不是继续换 backbone，而是以下几条：

1. `Task6 -> Task7` 是否稳定优于直接 `Task7`
2. 多图病例恢复双图聚合或切面优先
3. 蓝板颜色标准化
4. 类别不平衡训练策略
5. 尺寸信息作为结构化特征接入

如果这五条中有两到三条有效叠加，`Task7` 的临床二分类结果有希望继续显著上抬。

## 15. 最终原则

这轮工作不追求“把所有可能性都试完”，而是追求：

- 先把最有临床意义的二分类做好
- 先减少高危漏诊
- 先把医生提的建议真正变成可验证的实验
- 再决定哪些值得长期保留为论文主线

换句话说：

- 研究上，`Task6` 仍然重要
- 但临床主线，后面要更明确地围绕 `Task7`

这就是这一轮冲刺计划的核心。
