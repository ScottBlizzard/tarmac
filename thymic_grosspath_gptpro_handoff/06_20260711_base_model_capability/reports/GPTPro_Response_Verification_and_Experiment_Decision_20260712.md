# GPT Pro 回复核验与实验决策

日期：2026-07-12  
输入：`汇报/七月十二gptpro.md`  
主任务：Task7 胸腺大体图像低危/高危全覆盖分类与跨来源泛化

## 1. 执行结论

GPT Pro 的核心审计成立，而且比此前内部重构报告发现的问题更严重：

1. No.64 的 0.9263 不是无泄漏性能。获胜路由器输入包含由真实标签衍生的 `difficulty`，整个上游候选、路由、纠偏和预算选择也未在完整流水线层面严格嵌套。
2. Candidate41 的 0.8351 不是干净的视觉模型融合。获胜 stacker 只读取四套既有纠偏系统的预测、概率、置信度、投票和分歧。
3. `base162` 的 nominal `holdout234` 在元堆叠阶段与 adapt72 合并并重新分折，已不再是独立留出；完整 OOF 上选择阈值、全矩阵预处理以及来源字段又带来额外乐观偏差。
4. C1/C2 是当前最接近“图像本身能力”的候选，但其可信内部水平仍约为 OOF BAcc 0.748-0.751、source-LODO BAcc 0.740-0.744，尚未证明外部泛化突破。
5. 下一轮第一优先级应是 GPT Pro 路线 A1：利用原始高分辨率图像建立全局到局部的直接分类器。只有图像-only 的直接模型出现增益后，才进入路线 B 的真实粗到细级联。

因此，项目当前不能再说“基础模型已经有 83.5% 或 92%”。最稳妥的表述是：

> 历史数据上存在较强的输出层纠偏信号，但它混有标签衍生泄漏和同数据反复选择，且没有迁移到外部来源；当前可辩护的图像驱动基线是 C1/C2，内部 OOF 与来源留出 BAcc 约 0.74-0.75。

## 2. 代码核验结果

### 2.1 No.64：确认存在直接标签衍生泄漏

证据链：

- `scripts/run_task7_concat_curriculum_probe.py` 用真实标签索引各模型的 `true_prob`，再据此生成 `correct_count`、`mean_true_prob`、`min_true_prob` 和难度分层。
- `scripts/run_task7_gross_hardcore_signal_20260521.py` 载入上述 difficulty 表，并在 `model_visible` 特征中拼接 `selection_rule` 和 `difficulty` 的哑变量。
- No.64 获胜路由器是 `best41_wrong_model_visible_extra`，因此获胜路径确实使用了该不可部署特征。
- `scripts/run_task7_image_only_hardcore_reviewer_20260521.py` 的获胜纠偏器为 `model_non_easy_extra`，最终特征集为 `model`，没有使用 DINO 图像 embedding。

结论：该结果不能通过“它是 OOF”来辩护。测试折上的 `difficulty` 本身已经偷看测试病例真实标签；而且上游候选和最终配置选择没有被完整包进最外层训练折。

### 2.2 Candidate41：确认是输出层 stacking

`scripts/run_task7_candidate_stacking_20260520.py` 的获胜配置是：

- 模型：`logreg_c03`；
- 特征集：`candidates_only`；
- 输入候选：29、36、38、39 四套既有自动纠偏系统；
- 特征：每套系统的最终预测、概率、置信度，以及候选投票比例、分歧、概率均值/方差/极值。

它没有重新读取图像，也没有直接组合独立视觉 backbone 的表征。0.8351 可保留为历史系统对照，但不能用于证明视觉表征已经达到 83.5%。

### 2.3 `base162`：确认 nominal holdout 被消耗

`scripts/run_task7_old_third_no64_meta_stack_20260523.py` 中：

1. 先载入 old、adapt72 和 holdout234；
2. 将 adapt 与 holdout 合并成完整 third_batch；
3. 对完整 third_batch 重新生成 StratifiedKFold；
4. 再与 old 合并训练元堆叠。

因此 `holdout234` 在这个阶段已参与交叉训练和模型选择。其他风险包括：在完整 OOF 上选决策阈值并在同一 OOF 报告；可选 DINO 缩放/PCA 在 CV 前拟合完整矩阵；加入 `domain` 和 `third_split` 哑变量，容易学习来源行为签名。

### 2.4 C1/C2：目前可保留，但只能称内部候选

C1/C2 直接从当前病例图像的确定性视图提取视觉特征，未发现像 `difficulty` 这样的真实标签推理特征。它们仍有两个边界：

- 候选是在同一 591 例内部实验中选出的，数值仍是模型开发估计；
- 历史 108 与 162 外部集已经看过标签，只能作压力测试，不能再作确认性外部验证。

## 3. 原始分辨率可行性核验

服务器只做读取统计，没有把图像下载到本地，也没有启动训练。主注册表共有 861 例：old_data 285、third_batch 306、strict_external 108、new_external_160 162。

原图尺寸：

| 数据域 | n | 原图主要尺寸 | 说明 |
| --- | ---: | --- | --- |
| old_data | 285 | 4608x3456 或 6000x4000 | 高分辨率原图可用 |
| third_batch | 306 | 4608x3456 或 6000x4000 | 高分辨率原图可用 |
| strict_external | 108 | 640x480 至 6000x4000 | 分辨率强异质 |
| new_external_160 | 162 | 1360x768 至 3840x2816 | 多数低于域内原图 |

按现有标本框算法估算，标本框 25% 局部块的短边达到 512 像素的比例为：

| 数据域 | 比例 |
| --- | ---: |
| old_data | 217/285，76.1% |
| third_batch | 267/306，87.3% |
| strict_external | 66/108，61.1% |
| new_external_160 | 28/162，17.3% |

结论：A1 在 591 例内部开发集上确实能引入 C1 之前没有充分利用的原生局部细节，不是对 2048 缓存图的重复放大。但外部数据分辨率明显更低，必须把分辨率鲁棒性作为预设压力测试，不能让模型把高清程度当作来源或类别线索。

## 4. 实验决策

### 4.1 主线先跑 A1，而不是先重建级联

第一实验建立全覆盖直接视觉分类器：

- 输入必须来自注册表中的原始 `image_path`，不能使用 max2048 缓存作为“原生细节”；
- 每病例固定 14 个视图：整图 1、标本 crop 1、覆盖标本的 50% 中尺度块 4、25% 局部块 8；
- 局部块按固定组织网格生成，不得根据标签、模型置信度、错误病例或来源选择；
- backbone 首轮固定 C1 的 SigLIP-L@512，不再同时更换编码器；
- 主任务只做 Task7 低危/高危，六分类辅助头暂不进入第一轮；
- 比较简单层级 gated MIL 与一个全局到局部 cross-attention 变体；
- 同时保留旧 C1 六视图聚合器、随机组织块和无全局上下文消融。

这一步先回答最重要的问题：同一 backbone 获得真正更细的像素信息后，能否在 100% 覆盖下直接提高分类能力。

### 4.2 分辨率鲁棒性必须预先固定

为了避免高清来源捷径，A1 必须同时报告：

- 按原图短边和 25% tile 有效像素分层的 OOF/LODO 指标；
- 在外层测试折上对高分辨率图做固定降采样再提取 tile 的压力测试；
- `native-detail` 与 `resolution-matched` 两套测试输入的配对差值；
- 不把像素尺寸、来源、设备或数据批次字段输入诊断头。

若 A1 的增益只存在于高清 old/third，而在降采样压力测试或整来源留出消失，应解释为采集分辨率依赖，不得晋级。

### 4.3 只有 A1 有信号后才跑真实级联 B1

Stage 1 固定 C1；路由固定为外层训练折中 C1 绝对 logit margin 最小的 40%，不扫描连续阈值。Stage 2 只读取原图固定局部块，不读取 C1 概率、置信度、来源、difficulty 或错误标签。

同一批 routed cases 比较：

| 方案 | 输入与作用 |
| --- | --- |
| M0 | 保留 C1，作为同病例基线 |
| M1 | 仅行为特征的固定 logistic，量化旧式纠偏 |
| M2 | 仅局部图像专家，检验新增视觉证据 |
| M3 | 图像专家加 C1 行为信息，且融合选择完全嵌套 |
| M4 | 与真实路由在类别/来源/基线错误率上匹配的随机路由 |

核心因果量是 routed subset 上 `BAcc(M2)-BAcc(M0)`，不是从很多门槛中挑出的最终总分。报告配对 bootstrap、置换或 McNemar、rescue/harm、B1/B2及来源分层；真实路由还必须超过 1,000 次匹配随机路由的第 95 百分位。

### 4.4 医生 ROI 与真实多视图是后续信息增量路线

- 先做约 120 例、六亚型各 20 例的盲法切面/ROI 标注 oracle；医生不知道标签和模型结果。
- 若人工 ROI 相对自动标本块没有明确增益，停止训练自动 ROI 网络。
- 当前只有约 17 个双图病例，不足以训练多视图模型；应把标准化整图、切面近景、包膜/界面视图纳入前瞻采集。
- 医生特征表继续用于错误归因、抽样和采集设计，不作为当前模型训练或推理输入。

## 5. 评估与晋级门槛

所有模型以患者为单位，外层测试折完全冻结；内层三折完成架构、正则和融合选择。除五折 OOF 外，必须做 batch1、batch2、third_batch 整来源轮流留出的 source-LODO。

### A1 直接模型晋级

- 相对 C1 的 OOF BAcc 至少 +0.03；
- 患者配对差值 95% CI 下界大于 0；
- source-LODO BAcc 至少 +0.02；
- 三个来源至少两个方向为正，且最低来源不得明显坍塌；
- 第二随机种子同方向；
- 增益不能只出现在 native 高清输入而在 resolution-matched 压力测试消失。

### B1 级联晋级

- routed subset 上 `M2-M0` BAcc 至少 +0.06；
- `M3-M1` 至少 +0.03，证明图像信息超过行为纠偏；
- 全量 OOF 至少 +0.02；
- source-LODO 至少 +0.015；
- B1 与 B2 不能通过牺牲其中一类制造总增益。

候选预算固定为三个真正改变输入信息的家族，每家最多两个变体：原生局部块、医生 ROI、真实额外视图。若均未通过门槛，停止在同一 591 例上继续搜索 head/loss/weight，转向标准化采集和新增病例。

## 6. 执行顺序

1. 修正全部报告和模型排行榜口径，退役 No.64 的 0.9263 前瞻解释。
2. 运行一个廉价、完整嵌套、移除 `difficulty` 的 No.64 重建，仅用于确定历史纠偏剩余量；不得阻塞视觉主线。
3. 运行 A1 原生细节直接分类 pilot，先比较旧 C1 聚合、native tile 简单 MIL、坐标/尺度层级聚合。
4. 只有 pilot 达到预设方向后，补全严格嵌套、所有 source-LODO、分辨率压力测试和第二种子。
5. A1 通过后运行固定路由 M0-M4；未通过则优先医生 ROI oracle，而不是继续调路由。
6. 最终最多锁定 C1、C2 和一个新模型，进入全新多中心盲测。历史 108/162 不参与选择。

## 7. 计算与存储决定

当前服务器根盘只剩约 17 GB，而预计 14 视图 SigLIP-L dense fp16 token bank 约 17.3 GB，直接落盘风险过高。服务器可用内存约 106 GB，因此首轮采用以下方式：

- 从原图流式提取 14 视图 frozen tokens；
- token bank 暂存在进程 RAM；
- 同一作业中训练各外层折 head；
- 磁盘只保存病例级 OOF/LODO 预测、配置、日志和指标；
- 不把服务器图像或大型特征库下载到本地。

这既满足原生图像信息要求，也避免再次占满根盘或产生下载费用。
