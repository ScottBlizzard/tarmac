# Task7 强骨干六视图 Wave C 预注册

日期：2026-07-12

## 目标

继续提升 100% 覆盖、无拒识、无人工复核的纯图像 Task7 基础能力。该实验只回答：在 C1 已证明有效的六视图输入和 dense-token gated 聚合协议下，具有不同归纳偏置的未测试预训练表征是否能稳定超过 C1。

## 固定候选

1. `SwinV2-L@384`：层级局部窗口注意力。
2. `ConvNeXtV2-L@384`：卷积结构与 FCMAE 预训练。
3. `SigLIP-SO400M@512`：检验模型容量与 512 分辨率是否存在交互；既往 `SO400M@378 whole/crop` 的失败仍保留，不据此调整本轮参数。

每个骨干固定使用 `whole/crop/crop_q0/crop_q1/crop_q2/crop_q3` 六视图、全部 dense token、同一个低容量 gated 聚合头、类别加权交叉熵、阈值 0.5 和种子 20260712。每个骨干只运行一个头，不搜索 pooling、损失、阈值和融合权重。

## 评估

- 病例级固定五折 OOF。
- `batch1 / batch2 / third_batch` 三来源整批留一。
- 主指标：BAcc、AUC、Sensitivity、Specificity、最低来源 BAcc、B1/B2 风险准确率。
- 与锁定 C1 做病例级配对 bootstrap，并报告每来源方向。
- 严格外部 108 例和已查看的新外部 162 例不参与训练、选模或阈值。

## 晋级门槛

候选只有同时满足以下条件才进入第二种子和固定等权互补验证：

- 五折 OOF BAcc 不低于 C1，且点差至少 `+0.015`；
- source-LODO BAcc 点差至少 `+0.015`；
- 至少两个留出来源方向为正；
- 任一来源 BAcc 不比 C1 下降超过 `0.02`；
- Sensitivity 和 Specificity 均不低于 `0.70`；
- B1 与 B2 不出现一方提升、另一方明显恶化的不可接受交换。

不满足门槛即关闭该骨干六视图分支，不追加普通超参数搜索。通过者仍只能称为内部候选，必须等待全新独立外部数据确认。

## 存储边界

三个特征库顺序生成。每个骨干完成 OOF 和 source-LODO 后，归档 metadata、配置和 SHA-256，删除可再生的 dense 特征数组；保留病例级预测、模型头、日志和指标。任何患者图像或特征库都不从服务器下载到本地。
