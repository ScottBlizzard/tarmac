# H12 PE 最后块 LoRA 表征适配预注册

日期：2026-07-15

## 1. 改变的假设

H11 已经在锁定 H10/H3 模型上排除了一条具体路线：紧裁、背景中和、尺度归一化和无标签局部证据窗口均不能稳定恢复关键错误。继续改变同一冻结 PE token 的取景或浅层汇聚方式，没有产生新的视觉语义。

H12 只改变一个此前在 PE-Spatial 上没有直接检验的假设：

> PE-Spatial 前 23 个视觉块已经保留了局部形态，但冻结的最后一个 Transformer 块没有把这些 token 组织成适合本项目 B1/B2/B3 风险边界的关系；受控、低秩的最后块适配可能提高真实图像分类，而不需要置信度门控、医生文字或新模型下载。

H12 是当前固定 591 例上的探索性表征适配，不是独立确认性研究。若 source-LODO 不通过，本方向终止，不继续搜索 LoRA rank、层数、学习率、loss、阈值、seed 或融合。

## 2. 锁定数据与参考

- 队列：591 例，batch1 117、batch2 168、third_batch 306。
- 亚型：A 44、AB 262、B1 62、B2 89、B3 24、TC 110。
- 二分类：A/AB/B1 为低风险，B2/B3/TC 为高风险。
- 六个固定视图：`whole, crop, crop_q0, crop_q1, crop_q2, crop_q3`。
- 固定阈值：`0.5`；覆盖率必须为 591/591。
- 主评价：三来源 source-LODO；普通五折仅在 source-LODO 全部通过后运行。
- 参考模型：锁定 H3 PE-Spatial 六视图 dense gated head。

H3 source-LODO 参考：BAcc 0.7539、敏感度 0.6816、特异度 0.8261、B1 40/62、B2 52/89、third-B2 13/29。

H3 五折参考：BAcc 0.8003、敏感度 0.8072、特异度 0.7935、B1 42/62、B2 62/89。

## 3. 中间 token 缓存

不在每个 epoch 重跑 3.04 亿参数编码器。使用锁定 PE 权重逐视图执行：

```text
forward_features(layer_idx=-2, norm=False, strip_cls_token=False)
```

PE 共 24 个块；该调用完成 0--22 号块，缓存进入第 23 号最终块之前、仍含 CLS 的 token。每个视图最多 1024 个 patch token 加 1 个 CLS，缓存为 float16：

```text
[591, 6, 1025, 1024]
```

空间网格单独保存。最终块后删除 CLS，再恢复为 H3 的 `[6, 1024, 1024]` dense 输入和有效 mask。

工程验证已在任何性能训练前完成：

- 不量化的前缀经过最终块和 `ln_post` 后与完整 24 块规范化前向逐元素最大误差 `0.0`；
- float16 前缀在一个六视图病例上的锁定 H3 概率误差约 `4.95e-6`；
- 最终块单视图反向约 `0.0034 s`；
- 不需要下载任何权重或图像。

## 4. 锁定模型

### 4.1 冻结部分

- PE-Spatial-L14-448 patch embedding、位置编码、0--22 号 Transformer 块全部冻结。
- 第 23 号块原始参数全部冻结。
- 第 23 号块之后、H3 dense token 使用的 `ln_post` 全部冻结并原样执行。
- 医生概念、医生文字、H10 难度、H11 视觉归因、历史概率和来源 ID 均不作为模型输入。

### 4.2 LoRA 位置

只在第 23 号块四个二维权重上加入低秩增量：

1. attention `in_proj_weight`；
2. attention `out_proj.weight`；
3. MLP `c_fc.weight`；
4. MLP `c_proj.weight`。

所有位置固定 `rank=8, alpha=8`，即缩放为 1。A 矩阵 Kaiming 初始化，B 矩阵全零初始化，因此 epoch 0 的视觉块严格等于预训练最终块。LoRA 参数共 131,072。

### 4.3 分类头

- 使用原 H3 `MaskedDenseGatedHead`：hidden 128、attention 64、dropout 0.10。
- 每个外层折从该折锁定 H3 `best_head.pt` 初始化，而不是随机初始化。
- 分类头 151,107 个参数可训练。
- 总可训练参数 282,179；不训练来源路由器、拒识器或第二阶段概率模型。

## 5. 复现硬闸门

训练前，对 source-LODO 三个测试来源分别加载对应 H3 头，保持 LoRA 零增量，从前缀缓存恢复最终 token 并推理。

只有同时满足以下条件才开始任何适配训练：

1. 591/591 个固定阈值分类与锁定 H3 OOF 一致；
2. 最大 `prob_high` 绝对误差 `<=0.005`；
3. 病例、标签、来源和外层折一一对应。

失败时只排查缓存与数值路径，不放宽闸门，不训练。

## 6. 锁定训练

- Seed：`20260715`。
- Source-LODO：每次完整留出一个来源；验证集沿用 H3 的预定下一 master fold；测试来源不参与梯度、早停或阈值。
- 训练采样：与 H3 相同的来源×风险逆频率采样。
- 损失：仅二分类 cross-entropy。
- Batch：物理 batch 4，梯度累积 2，等效 batch 8。
- Epoch：最多 24；patience 5。
- Optimizer：AdamW。
- LoRA 学习率 `1e-4`，weight decay `1e-2`。
- H3 头学习率 `3e-5`，weight decay `1e-4`。
- 梯度裁剪：1.0。
- 无 scheduler、无标签平滑、无 class weight、无辅助六分类、无概念 loss、无增强搜索。
- 选择指标：验证 BAcc；完全相同时依次用验证 AUC、较早 epoch 决胜。
- Epoch 0 的未适配 H3 状态必须进入候选，因此某折可以诚实选择“不适配”。

## 7. Source-LODO GO / NO-GO

以下条件必须全部通过：

| 条件 | 要求 |
| --- | ---: |
| 覆盖/阈值 | 591/591，固定 0.5 |
| pooled LODO BAcc | >= 0.7639，即至少比 H3 +0.0100 |
| 高风险正确数 | >= 157/223，H3 为 152/223 |
| 低风险正确数 | >= 299/368，H3 为 304/368 |
| B1 | >= 40/62，不低于 H3 |
| B2 | >= 56/89，至少比 H3 多 4 例 |
| third-B2 | >= 17/29，至少比 H3 多 4 例 |
| 来源方向 | 至少 2/3 来源 BAcc 高于 H3 |
| 最差来源 | BAcc >= 0.7381，且任何来源相对 H3 下降不超过 0.015 |
| 配对 bootstrap | 平均 ΔBAcc > 0；P(Δ>0) >= 0.80；95% CI 下界 > -0.010 |

该组合刻意要求 B1 和 B2 不再一升一降。只提高 A/AB/TC、只提高特异度、只改变概率校准或只在一个来源获益均为 NO-GO。

## 8. 条件性五折与确认 seed

只有 source-LODO 全通过才运行相同配置的普通五折。五折必须同时满足：

- BAcc >= 0.8053；
- 敏感度和特异度均 >= 0.7900；
- B1 >= 42/62；
- B2 >= 62/89；
- 五个外层折均不低于 0.70 BAcc；
- 相对 H3 的配对 ΔBAcc 为正，95% CI 下界 > -0.010。

只有 source-LODO 和五折都通过，才允许 seed `20260716` 完全原样确认。确认 seed 必须独立通过所有点估计门槛，两 seed 平均也必须通过。

## 9. 停止规则

- 复现失败：停止，修复工程路径后原样重跑。
- Source-LODO 任一科学 gate 失败：H12 NO-GO；不运行五折、不读外部队列、不搜索其他 LoRA 配置。
- Source-LODO 通过、五折失败：H12 NO-GO；不运行确认 seed。
- 两阶段通过、确认 seed 失败：H12 NO-GO。
- 全部通过：冻结候选，再讨论严格外部评估；不得根据外部结果回调模型。

- 新缓存预计约 7.3 GiB，可再生，实验结束后记录哈希并按存储需要删除。
- Git 只提交代码、哈希、预注册和聚合结果。
