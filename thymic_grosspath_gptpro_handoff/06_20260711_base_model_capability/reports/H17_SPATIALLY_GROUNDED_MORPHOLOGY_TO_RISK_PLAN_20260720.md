# H17 空间锚定形态语义到风险判别计划

日期：2026-07-20

状态：`PREREGISTRATION / NOT STARTED`

主线任务：在 591 例域内数据上，提高全覆盖、固定阈值 `0.5` 的低危/高危直接图像分类能力，并为 source-LODO 和严格外部泛化建立可信候选。

方案代号：`H17-SGMR`（Spatially Grounded Morphology-to-Risk）

## 0. 不可偏离的目标

H17 的目标不是提高“自动放行比例”，也不是重新包装两阶段 workflow，而是提高模型对每一例图像本身的判别能力。

最终模型必须同时满足：

1. 输入只有当前病例原始大体图像；
2. 591 例全部输出低危/高危预测，覆盖率为 100%；
3. 分类阈值固定为 `0.5`；
4. 不使用拒识、复核、置信度门控、阈值扫描或概率校准来制造准确率；
5. 不把来源、批次、亚型真值、医生文字记录或旧模型输出作为推理输入；
6. 多分支或多专家可以使用，但每个分支必须重新读取原始像素并学习明确的可见形态，不能只修正其他模型的置信度；
7. 域内三批数据仍作为一个 591 例队列进行普通五折训练，来源字段只用于审计和 source-LODO；
8. 严格外部数据保持封存，只有内部晋级和 source-LODO 均通过后才允许读取；
9. 外部结果无论好坏都不允许反向修改模型、阈值或候选选择。

## 1. H10 至 H16 的证据结论

### 1.1 当前真实基线

H10 是当前最好的全覆盖域内基础模型：

| 指标 | H10 |
|---|---:|
| Accuracy | 0.7986 |
| Balanced Accuracy | 0.7888 |
| AUC | 0.8430 |
| Sensitivity | 0.7489 |
| Specificity | 0.8288 |
| 判对数 | 472/591 |

分亚型判对数：

| 亚型 | N | H10 判对 |
|---|---:|---:|
| A | 44 | 34 |
| AB | 262 | 234 |
| B1 | 62 | 37 |
| B2 | 89 | 54 |
| B3 | 24 | 19 |
| TC | 110 | 94 |

因此，当前模型本身不是九十多准确率。九十多来自选择性放行与复核 workflow，不代表全量直接分类能力。

### 1.2 已经被实验排除的简单解释

H10 至 H16 已经系统测试并否定以下主要假设：

- 只要重新定义 easy/medium/hard 课程顺序就能提升；
- 只要更重视显著局部区域就能提升；
- 只要改变裁剪范围、背景、尺度或证据窗口就能提升；
- 只要使用二阶纹理就能获得稳定跨来源提升；
- 只要加入显式坐标、真网格或局部上下文关系就能提升；
- 只要让 PE-Spatial 最后一层进行 LoRA 适配就能提升；
- 只要使用病灶遮挡、局部充分性或完整性损失就能提升；
- 只要增加关系预训练、窗口数量、loss 权重或随机种子就能提升。

这不代表所有骨干、局部模型和多模型均无效，而是说明在仍然只有病例级低危/高危标签时，继续做同类结构和超参数搜索的边际证据价值很低。

### 1.3 H16 最关键的机制结果

H16 的核心结果不是“模型没有找到区域”，而是：

1. C3 对真实局部与错配上下文的区分 AUC 为 `0.9774`；
2. 空间顺序方向在 5/5 折均正确；
3. relation mismatch 的 correct-logit drop 从 C2 的 `0.032` 提升到 `1.396`；
4. match probability gap 从 `0.0056` 提升到 `0.7426`；
5. 但高危病灶方向均值为 `-0.249`，只有 1/5 折方向正确；
6. C3 的 BAcc 为 `0.7483`，相对 H10 下降 `0.0406`；
7. 配对 bootstrap 的 95% CI 为 `[-0.0761, -0.0050]`，`P(delta>0)=0.0129`。

因此，C3 确实学会了“哪个局部属于哪个病例、局部与上下文是否一致”，但没有学会“哪些局部形态以及哪些空间关系支持高危或低危”。关系学习越强，错误的风险方向反而被放大。

### 1.4 错误病例揭示的底层矛盾

H10/C2/C3 共同持续错误 65 例：

| 亚型 | 持续错误数 |
|---|---:|
| A | 6 |
| AB | 9 |
| B1 | 15 |
| B2 | 22 |
| B3 | 3 |
| TC | 10 |

H10、A1、A2、C2、C3 五个模型共同错误 49 例。

既往盲态视觉分析和医生记录分析显示，这些错误不是同一种困难：

1. **单张照片信息不足或不可见**：决定性切面、包膜界面、邻近组织关系没有被拍到；
2. **大体表型与组织学风险冲突**：低危病例可以呈暗红、出血样或异质，高危病例也可以呈苍白、均质、界限清楚；
3. **线索可见但模型解释错误**：模型注意到显著区域，却把非特异性颜色、纹理或破损当成风险证据；
4. **区域关系缺失**：同样的暗区如果位于内部、边缘或跨越包膜，其意义可能不同；
5. **标签监督粒度错误**：一张图只有一个风险标签，没有告诉模型哪个区域、哪种形态、哪组关系产生该标签。

13 例持续困难病例的人工盲态根因分析中，约 4 例属于信息不足或歧义，约 6 例属于大体表型与风险标签冲突，只有约 3 例可明确归为“图中有可恢复线索但模型读错”。这说明当前 591 例单图数据存在真实上限，不能把全部 119 个错误都假设成可由更强分类头修复。

### 1.5 医生文字记录能做什么，不能做什么

医生记录覆盖约 589 例，但它不是当前图像的完整视觉标注：

- 记录可能来自触诊、多切面或术中观察，而当前模型通常只有一张照片；
- “未提及”不能当作“明确不存在”；
- 359/589 的记录同时包含名义上的低危和高危概念；
- 单个文字概念对风险的 AUC 约为 `0.6621`；
- C1/C2 有相对完整的正负记录，但其他概念大量只有阳性、没有可靠阴性；
- C4“均质苍白切面”在现有文字中无法形成可训练标签。

所以医生记录继续保持以下边界：

1. 不作为风险模型输入；
2. 不把“未提及”编码为阴性；
3. 只用于定义形态词典、分析错误机制、设计标注规范；
4. 现有明确阳性/阴性记录最多只能用于一个独立的概念可辨识性审计，不得进入最终风险训练或推理。

## 2. H17 的核心科学假设

### 2.1 主假设

如果先让模型在图像中学习一组**空间位置明确、可重复标注、与大体形态直接对应**的中间概念，再让模型学习这些概念之间的组合如何支持低危/高危判断，那么模型可能纠正 H16 中“关系学会了、风险方向学反了”的问题。

该假设的因果链为：

```text
原始像素
  -> 解剖/标本区域定位
  -> 区域内可见形态概念
  -> 跨区域关系
  -> 概念组合与残余视觉证据
  -> 低危/高危全覆盖预测
```

H17 不假设某一个概念天然单调对应高危。例如，出血、暗红、异质和界限清楚都可能跨风险组出现。风险应来自多个概念及其空间关系的组合。

### 2.2 可证伪预测

若主假设正确，应同时看到：

1. 至少 4 个关键形态概念能在留出病例中被模型可靠预测；
2. 概念预测能跨三来源保持基本稳定；
3. 正确的 ROI 比随机 ROI、错位 ROI 和标签置乱显著更有效；
4. 去除关键区域或干预关键概念时，风险 logit 按预期方向变化；
5. 风险提升不仅发生在已标注病例，也发生在没有概念标注的留出病例；
6. B1/B2 边界获得净改善，而不是只提高 AB 或 TC；
7. 改善在至少 3/5 折、多个来源和确认种子中重复。

若上述链条中任一关键环节失败，应停止该分支，而不是继续调参。

## 3. H17 明确不做的事情

以下做法不属于 H17：

- 不重新搜索拒识阈值、放行比例或复核规则；
- 不用 H10/H16 的概率、置信度、错误标记或预测类别作为新模型输入；
- 不按 batch1、batch2、third_batch 分别建模后再调权重；
- 不用严格外部数据选概念、选候选或选阈值；
- 不把医生风险判断、医生置信度或“模型是否判错”重新收集一遍；
- 不把医生记录直接拼接到图像特征中；
- 不把自动显著区域当作病理语义真值；
- 不把未知概念当作阴性；
- 不继续开展同类 LoRA rank、局部窗口个数、loss 权重和 seed 大搜索；
- 不用亚型真值作为推理路由；
- 不以普通 Accuracy 单独晋级，因为 AB 数量过大时 Accuracy 会掩盖高危敏感度下降。

## 4. 形态语义体系

### 4.1 区域角色

每个区域都必须来自当前照片可见像素：

| 代码 | 区域角色 | 说明 |
|---|---|---|
| Z0 | 全局标本 | 整体形状、比例、完整性和拍摄质量 |
| Z1 | 主切面 | 肿物切面主体，不等同于整张图 |
| Z2 | 外周包膜/边界界面 | 肿物与周围组织交界处 |
| Z3 | 内部异常区 | 出血、坏死、囊变、软化、结节或明显异质区域 |
| Z4 | 邻近组织界面 | 脂肪、胸腺组织或其他邻近组织与肿物连续关系 |
| Z5 | 伪影区 | 阴影、反光、血液污染、器械遮挡、背景等非病理区域 |

标注形式优先使用粗多边形或框，不要求像素级分割。若区域在当前图像不可见，必须标记 `not_visible`，不能猜测。

### 4.2 视觉概念

所有概念均采用 `present / absent / not_visible_or_uncertain` 三态：

| 代码 | 概念 |
|---|---|
| M1 | 包膜或外周边界完整、连续、相对清楚 |
| M2 | 包膜中断、边界模糊、不规则或疑似跨界连续 |
| M3 | 主切面整体偏苍白且相对均质 |
| M4 | 主切面明显异质、斑驳或不同质地区域并存 |
| M5 | 明确出血、坏死、囊变或软化样内部异常 |
| M6 | 分叶、结节、多结节或纤维隔样结构 |
| M7 | 异常组织与邻近组织连续、黏连或疑似受累 |
| M8 | 内部异常区主要局限于肿物内部，未到达外周界面 |
| M9 | 影响判断的阴影、反光、血液覆盖、破损或器械伪影 |
| M10 | 当前图像存在有效主切面，可用于切面形态判断 |

M9 是负控制概念。模型若主要依靠 M9 提高风险分类，应视为伪相关，而不是成功。

### 4.3 空间关系

只预注册少量可解释关系，不开放任意 attention 关系搜索：

| 代码 | 关系 |
|---|---|
| R1 | 内部异常区是否被完整边界包围 |
| R2 | 异质或异常区位于内部、外周，还是跨越外周界面 |
| R3 | 肿物异常区是否与邻近组织形成连续通路 |
| R4 | 分叶/结节/纤维隔与异质区是否空间对应 |
| R5 | 明显内部损伤与完整包膜是否同时存在 |
| R6 | 苍白均质切面与完整边界是否同时存在 |

这些关系只描述图像事实，不由标注者给出“高危”或“低危”方向。风险方向由训练折内的风险标签学习，并通过机制消融验证。

## 6. 阶段 H17-0：零新增风险诊断的可行性审计

### 6.1 目的

在投入任何新形态标注前，先回答两个问题：

1. 当前照片中是否能稳定识别“包膜/边界清楚”和“包膜不完整/边界不清”这两个最完整的历史概念；
2. 现有自动区域能否定位到与这些概念相关的位置，而不是背景或颜色伪影。

### 6.2 历史记录的使用边界

只允许使用 C1/C2 中**明确写出**的阳性和明确否定：

- C1：356 阳性、89 可靠阴性、146 未知；
- C2：89 阳性、356 可靠阴性、146 未知。

未知 146 例完全从概念 loss 和概念指标中排除。

此阶段只训练冻结视觉特征上的轻量审计 probe，输出不得进入 H17 风险模型，不得被当作医生真值替代品。它只决定是否值得开展图像形态标注。

### 6.3 审计候选

1. `P0-global`：冻结 PE-Spatial 全局特征加线性概念头；
2. `P1-spatial`：冻结 token 特征加 attention pooling 概念头；
3. `P2-random-control`：概念标签在训练折内置乱；
4. `P3-source-control`：仅预测来源，检查概念特征是否被来源替代。

不训练风险头，不做骨干微调。

### 6.4 H17-0 判读规则

若同时满足以下条件，说明历史记录中至少有一部分边界语义可以在当前照片中复现，可直接进入 60 例 morphology-only pilot：

- C1/C2 留出折 macro AUC `>=0.75`；
- C1/C2 留出折 macro BAcc `>=0.68`；
- 至少 4/5 折 AUC `>0.70`；
- 三个来源各自 macro AUC `>=0.65`；
- 真标签相对置乱标签 AUC 至少高 `0.10`；
- 概念预测与来源预测不能近似等价；
- 显著区域至少主要落在标本或边界，不以背景为主。

若失败，只能说明“历史文字记录不能作为当前照片的概念监督”，不能直接证明当前照片没有可标注形态。此时：

1. 不得把 C1/C2 强行加入风险 loss；
2. 先执行 20 例 morphology-only 可见性 mini-pilot；
3. 20 例中若至少 4 个概念在 `>=70%` 图像可判定，并达到一致性门槛，再扩展到 60 例；
4. 若 20 例仍无法形成至少 4 个可靠概念，才停止当前单图概念路线并进入“新视图/新信息”分支。

这一步避免把“医生记录来自触诊或其他切面”误判为“当前图像中所有形态都不可学”。

## 7. 阶段 H17-A：最小形态标注 pilot

### 7.1 为什么需要这一阶段

H16 已证明病例级风险标签可以教会模型空间匹配，却不能告诉模型具体形态及其风险方向。若不增加新的空间语义监督，继续使用相同 591 个病例级标签训练更复杂结构，极可能继续学习非特异颜色、纹理和来源差异。

这里不是让医生重复判断低危/高危，也不是请医生核验模型是否判错。标注任务只回答“当前照片中看到了什么、在哪里、是否看不见”。

### 7.2 既有 120 例 packet 的处理

服务器既有目录：

`/workspace/thymic_project/experiments/base_model_capability_20260711/physician_roi_oracle_120_20260712`

已核验两个 reader 表格均为 `pending`，没有完成标注。旧表包含风险判断和置信度字段，不应直接继续使用。

H17 必须重新生成 morphology-only packet：

- 删除风险判断；
- 删除风险置信度；
- 删除“模型是否判错”；
- 删除旧模型预测和置信度展示；
- 只保留 Z0-Z5、M1-M10、R1-R6 和图像可见性；
- 标注者对病例风险标签、亚型、来源和模型结果保持盲态。

### 7.3 Pilot 抽样

第一阶段只标 60 例：

- 每个亚型 10 例；
- 低危/高危各 30 例；
- 每个亚型内尽量覆盖三个来源；
- 在来源和亚型内覆盖清晰度、构图和可见主切面情况；
- 不根据 H10/H16 是否判错、置信度高低或 persistent-hard 身份抽样；
- 使用固定随机种子生成 manifest；
- 严格外部病例不得进入。

这样可避免只标旧模型错误病例造成适应性过拟合。

### 7.4 标注流程

推荐最小工作量方案：

1. 一名具有医学背景的项目成员完成 60 例第一遍形态标注；
2. 一名胸腺病理/外科专家仅对 60 例进行独立盲态复核，或至少复核其中 30 例；
3. 只对术语定义和明显分歧进行一次规则澄清；
4. 不要求专家重新给出风险类别；
5. 单例目标时间控制在 1 至 2 分钟；
6. 标注工具自动记录 `not_visible`，不允许强制二选一；
7. 所有修改保留版本和审计日志。

若医院不方便提供第二名完整标注者，可采用：

- 研究成员标 60 例；
- 医生盲态复核分层抽取的 20 例；
- 所有分歧概念在后续训练中先标为未知，不做强制裁决。

### 7.5 Pilot 质量门槛

满足以下条件才扩展到 120 例：

- 至少 4 个诊断形态概念在 `>=70%` 图像中可判定为 present 或 absent；
- 对这些概念，Gwet AC1 或 Cohen kappa `>=0.60`；
- 区域角色 Z1/Z2/Z3 的粗 ROI pointing agreement `>=0.60`；
- `not_visible` 的使用具有一致性，而不是一名标注者几乎不用；
- M9 伪影概念与病例风险不呈异常强相关；
- 医生认为概念定义对应当前图像可观察事实，而不是必须依靠触诊或额外切面。

若只有 2 至 3 个概念通过，不进入完整模型，先缩减词典并做一次 20 例规则复测。复测仍少于 4 个概念则停止。

### 7.6 扩展标注

Pilot 的可见性和下述形态风险信息量审计均通过后，才扩展到总计 120 例：

- 每个亚型最终 20 例；
- 新增 60 例仍不按模型错误或置信度抽样；
- 新增病例由一名标注者完成；
- 其中 20% 由第二标注者盲态复核；
- 分歧无法按规则解决时设为未知；
- 不为了平衡概念而把未知改成阴性；
- 不使用严格外部图像。

### 7.7 形态风险信息量与概念冲突审计 G0

“概念能被人看见”不等于“概念足以帮助低危/高危分类”。在扩展到 120 例之前，必须用 60 例 pilot 做一个只用于决策的 oracle 审计。

该审计不训练正式图像模型，不产生论文主结果，只回答：

1. 人工形态概念本身对风险是否有信息量；
2. 相同或近似概念组合是否大量对应相反风险；
3. 概念加入冻结视觉特征后，是否提供超出原始视觉 embedding 的增量。

预注册三个轻量模型：

1. `O0-concept-only`：带 L2 正则的 logistic regression，只使用 M1-M10 和 R1-R6；
2. `O1-frozen-visual`：冻结 PE-Spatial embedding 加同容量线性风险头；
3. `O2-visual-plus-concept`：冻结视觉 embedding 加人工概念，参数量与 O1 尽量匹配。

评估规则：

- 在 60 例内做重复分层五折，仅作为可行性估计；
- 所有缺失/不可见概念保留独立 mask，不填成 0；
- 只允许第 4.2 和 4.3 节预注册概念与关系；
- 不按测试折增加交互项；
- 同时统计完全相同概念向量和 Hamming 距离 `<=1` 的近邻风险冲突率；
- 与这 60 例对应的 H10 OOF 结果做配对参照；
- 报告 bootstrap 区间，不把小样本点估计当作正式结论。

进入 120 例扩展的 G0 门槛：

- `O0-concept-only` mean AUC `>=0.72` 且 mean BAcc `>=0.68`；
- `O2-visual-plus-concept` 相对 O1 的 mean BAcc 至少 `+0.03`，或 AUC 至少 `+0.03`；
- 至少 4 个概念具有跨折稳定的非零贡献；
- 近邻概念组合的风险一致率 `>=0.65`；
- 增量不能只来自 M9 伪影或来源差异；
- 在这 60 例的 H10 错误中，概念 oracle 至少显示出一组可解释的净 rescue，而不是只伤害原本正确病例。

G0 失败时，不扩展到 120 例，也不训练深层概念风险模型。此时的正确结论是：当前定义的可见形态虽然可能存在，但不足以提供稳定的风险增量，应转向增加视图或重新定义具有病理对应的形态关系。

## 8. 阶段 H17-B：概念可学习性实验

### 8.1 目标

这一阶段不看风险准确率是否提高，只验证：

> 模型能否从当前图像的像素中，在未见病例和不同来源中识别经过空间锚定的形态概念。

概念不可学习时，概念到风险模型没有科学基础。

### 8.2 五折规则

在每个外层折中：

- 训练折的 ROI/概念标签可以用于训练；
- 验证折只选择 checkpoint；
- 测试折概念标签只在 checkpoint 锁定后评估；
- 测试折 ROI 不得用于 proposal、prototype、loss 或阈值；
- 同一概念的判定阈值固定为 `0.5`，不按来源调节；
- 训练折内概念类别不平衡使用 capped inverse-frequency weight；
- 未知标签在 loss 中 mask。

### 8.3 模型候选

只运行以下预注册候选，不做开放式搜索：

#### B0：冻结全局概念 probe

- PE-Spatial 冻结；
- 全局特征预测 M1-M10；
- 不使用 ROI；
- 作用是下限和“只靠整体颜色/纹理”的对照。

#### B1：空间概念 embedding

- PE-Spatial token 冻结；
- 每个概念使用独立正/负 concept embedding；
- ROI 监督概念 attention 落在对应区域；
- 未知概念不参与 loss；
- M9 作为伪影负控制；
- 不训练风险头。

#### B2：空间概念 embedding 加关系头

- 在 B1 上预测 R1-R6；
- 关系只能读取对应区域 embedding；
- 不读取病例风险标签；
- 不允许任意全连接关系搜索。

#### 控制实验

- `B1-shuffled-label`：训练折内置乱概念标签；
- `B1-random-ROI`：使用面积和位置分布匹配的随机 ROI；
- `B1-wrong-role`：交换 Z2 与 Z3 的区域角色；
- `B1-source-only`：仅测试来源是否比概念更容易被编码。

### 8.4 概念指标

只在有明确 present/absent 标签的病例上计算：

- 每概念 AUC；
- 每概念 BAcc；
- sensitivity、specificity；
- macro AUC 和 macro BAcc；
- 三来源分层 AUC/BAcc；
- 五折稳定性；
- ROI pointing-game accuracy；
- soft IoU 或框命中率；
- 概念标签置乱差值；
- 随机 ROI 差值；
- concept-source mutual information 审计。

### 8.5 概念晋级门槛 G1

主模型 B1/B2 必须同时满足：

1. 至少 4 个非伪影概念的 OOF AUC `>=0.70`；
2. 通过概念的 macro AUC `>=0.78`；
3. 通过概念的 macro BAcc `>=0.70`；
4. 至少 4/5 折的 macro AUC `>=0.72`；
5. 每个来源的 macro AUC `>=0.68`；
6. ROI pointing-game accuracy `>=0.65`；
7. 真 ROI 相对随机 ROI 的 macro AUC 至少高 `0.05`；
8. 真标签相对置乱标签的 macro AUC 至少高 `0.10`；
9. M9 不能成为表现最强的“风险相关”概念；
10. 概念可见性和性能不应完全由单一来源驱动。

G1 失败时不训练风险模型。

## 9. 阶段 H17-C：形态到风险模型

### 9.1 主模型结构

推理时只输入原始图像。主模型由四类真实视觉专家组成：

1. **全局/主切面专家**：学习 M3、M4、M6、M10；
2. **包膜/边界专家**：学习 M1、M2；
3. **内部异常专家**：学习 M5、M8；
4. **邻近组织专家**：学习 M7；
5. **伪影控制头**：学习 M9，但 M9 不允许直接作为风险正向证据。

这些专家不是对置信度进行门控。每个专家都读取相应原始像素区域，并产生可单独验证的形态预测。

主模型输出由三部分构成：

```text
image-derived concept embeddings
  + preregistered relation embeddings
  + low-capacity residual visual embedding
  -> binary risk head
```

保留低容量 residual visual branch 的原因是现有概念词典不可能完整覆盖所有判别信息。硬概念瓶颈可能因概念不完整而人为限制模型上限。残差分支必须通过容量匹配和消融证明它没有完全绕过概念路径。

### 9.2 只允许的概念组合

风险头只显式建模以下组合：

- M1/M2 与 M5；
- M1/M2 与 M4；
- M2 与 M7；
- M5 与 M8；
- M4 与 M6；
- M3 与 M1；
- R1-R6。

不预设“某概念必然高危”。单概念主效应和组合效应均由训练折学习，但必须在五折中检查符号稳定性。

### 9.3 训练三阶段

#### C1：概念与定位预训练

- 初始化为 H10 使用的 PE-Spatial 权重；
- 首先冻结骨干；
- 只训练概念 embedding、定位头和关系头；
- loss 包含 masked tri-state concept loss、ROI localization loss、relation loss；
- 不读取风险标签；
- checkpoint 按验证折 concept macro AUC 和 localization 共同选择。

#### C2：冻结概念编码器，训练风险组合

- 冻结概念与定位模块；
- 训练 concept-composition risk head 和低容量 residual branch；
- 风险 recipe、采样和增强尽量与 H10 保持一致；
- 阈值固定 `0.5`；
- 不使用亚型或来源输入；
- checkpoint 只按训练折内预注册验证指标选择。

#### C3：受约束联合微调

仅在 C2 通过第一信号门槛后运行：

- 只解冻 PE-Spatial 最后一块或参数量匹配的 adapter；
- 同时保留 concept、localization、relation loss，防止概念遗忘；
- 若验证期 concept macro AUC 相对 C1 下降超过 `0.03`，立即 early stop；
- 不搜索 LoRA rank；
- 不允许风险 loss 把概念表示重新旋转成不可解释的病例捷径。

### 9.4 Loss 规范

所有 loss 先按训练折首个 epoch 的指数滑动均值归一化，再使用固定权重：

```text
L = 1.0 * L_risk
  + 1.0 * L_concept
  + 0.5 * L_localization
  + 0.5 * L_relation
  + 0.25 * L_concept_consistency
```

规则：

- 未知概念完全 mask；
- 概念类别权重使用 capped inverse-frequency，最大权重不超过 4；
- 风险类别采样沿用 H10，不为某个候选单独修改；
- 不使用旧模型置信度作为样本权重；
- 不使用 persistent-hard 身份作为样本权重；
- 不进行 loss 权重网格搜索；
- 只允许根据训练折数值稳定性做一次与候选无关的统一缩放修复。

### 9.5 数据增强

保留不会破坏形态语义的增强：

- 小角度旋转；
- 水平/垂直翻转；
- 轻度尺度和位置扰动；
- 轻度亮度与白平衡变化；
- JPEG 或传感器噪声模拟；
- ROI-aware crop，但必须保留被监督区域。

禁止或严格限制：

- 强颜色抖动把苍白/暗红意义破坏；
- CutMix/MixUp 把包膜、内部异常和邻近组织关系拼错；
- 随机裁掉全部主切面；
- 以背景替换制造新的来源捷径；
- 只对某一风险组使用特殊增强。

## 10. 风险候选与消融预算

只运行以下四个正式风险候选：

| 候选 | 结构 | 目的 |
|---|---|---|
| D0 | H10 原始基线复现 | 锁定参照 |
| D1 | 容量匹配的直接视觉模型，无概念监督 | 排除“参数更多”解释 |
| D2 | 硬概念 bottleneck，无 residual | 检查概念集合是否足够 |
| D3 | 概念 embedding + 关系 + 低容量 residual | H17 主候选 |

只在 D3 产生第一信号后运行机制消融：

| 消融 | 目的 |
|---|---|
| D3-no-ROI | 判断空间锚定是否必要 |
| D3-random-ROI | 排除额外裁剪本身的收益 |
| D3-no-relation | 判断关系组合是否必要 |
| D3-shuffled-concept | 排除多任务正则化假象 |
| D3-residual-only | 判断模型是否绕过概念路径 |
| D3-concept-only | 判断 residual 是否只在弥补概念不完整 |

正式候选只有一个开发种子。未通过第一信号门槛前，不运行确认种子，不启动新骨干。

## 11. 风险评估与晋级门槛

### 11.1 必报指标

所有 591 例 OOF、阈值 `0.5`、100% 覆盖：

- Accuracy；
- Balanced Accuracy；
- AUC；
- Sensitivity；
- Specificity；
- 混淆矩阵；
- 六亚型判对数；
- 三来源 BAcc；
- 五折 BAcc；
- 配对 bootstrap delta BAcc；
- `P(delta>0)`；
- McNemar 配对错误变化；
- H10 错误中的 rescue 数；
- H10 正确病例中的 harm 数；
- 65 例持续错误和 49 例五模型共同错误的变化；
- 有概念标注病例与无概念标注病例分开报告。

### 11.2 第一信号门槛 G2

D3 相对 H10 必须同时满足：

- BAcc `>=0.7988`，即至少 `+0.0100`；
- 配对 bootstrap `P(delta BAcc>0) >=0.90`；
- 至少 3/5 折 BAcc 高于 H10；
- Sensitivity `>=0.7489`；
- Specificity `>=0.8138`；
- B1 判对 `>=37/62`；
- B2 判对 `>=54/89`；
- 任一亚型判对数下降不超过 3；
- rescue 数大于 harm 数；
- 无概念标注病例上的 BAcc 也必须高于 H10；
- 至少一个空间/概念机制检验通过。

G2 失败即停止 D3，不用额外 seed 挽救。

### 11.3 内部晋级门槛 G3

进入确认种子前，D3 还必须达到：

- BAcc `>=0.810`；
- Sensitivity `>=0.770`；
- Specificity `>=0.820`；
- B1 判对 `>=40/62`；
- B2 判对 `>=58/89`；
- 三来源中最低 BAcc `>=0.750`；
- 任一来源相对 H10 下降不超过 `0.020`；
- 至少 4/5 折风险方向机制一致；
- D3 明显优于容量匹配 D1；
- D3 明显优于 shuffled-concept 和 random-ROI；
- 提升不能只来自 120 个有概念标签的病例。

### 11.4 确认种子门槛 G4

G3 通过后才运行两个锁定确认种子：

- 三个种子的平均 BAcc `>=0.810`；
- 每个种子 BAcc 均高于 H10；
- 每个种子的 Sensitivity 均不低于 `0.7489`；
- 至少两个种子满足 B1 `>=40`、B2 `>=58`；
- 合并配对 bootstrap 的 95% CI 下界不低于 `0`；
- 不允许按 seed 选择最好结果作为唯一报告。

## 12. 机制验证

H17 必须证明“学到形态”而不只是“又找到一个域内捷径”。

### 12.1 概念干预

在测试折对图像预测概念做最小干预：

- 遮挡或替换包膜区域；
- 遮挡内部异常区；
- 交换内部区与外周区的位置；
- 保留纹理但打乱空间关系；
- 保留全局颜色但移除局部结构。

检查：

- 概念 logit 是否按预期下降；
- 风险 logit 是否通过预注册组合发生变化；
- 低危和高危病例的变化是否方向一致；
- 关系干预是否比随机区域干预更强。

### 12.2 概念路径依赖

至少满足：

- shuffled-concept 使概念指标明显崩溃；
- random-ROI 比真 ROI 差；
- residual-only 不能复现 D3 全部收益；
- concept-only 能保留一部分收益；
- 冻结 concept encoder 后，风险头仍能跨来源使用概念；
- 联合微调后概念性能下降不超过 `0.03`。

### 12.3 防记忆检验

将病例分为：

1. 有 H17 概念标注；
2. 无 H17 概念标注；
3. 既往 persistent-hard；
4. 既往 H10 正确。

若收益只出现在第 1 组，不认为基础能力提高。主候选必须在第 2 组仍有正向 BAcc delta，并在第 4 组控制新增伤害。

## 13. Source-LODO 与严格外部

### 13.1 Source-LODO 门槛 G5

只有 G4 通过后运行三次 source-LODO：

- 留出 batch1；
- 留出 batch2；
- 留出 third_batch。

所有概念学习、ROI 原型、风险头和 checkpoint 选择只读取剩余来源。

晋级条件：

- mean source-LODO BAcc 至少比锁定的最佳既往视觉基线高 `0.010`；
- 任一留出来源 BAcc 相对对应基线下降不超过 `0.020`；
- 每个留出来源 Sensitivity 和 Specificity 均不低于 `0.68`；
- B1/B2 不出现单一来源崩溃；
- 概念 macro AUC 在留出来源仍 `>=0.68`；
- 不使用来源特定阈值。

### 13.2 严格外部

G5 通过后，只评估一个锁定的 D3 模型：

- 阈值固定 `0.5`；
- 100% 覆盖；
- 不校准；
- 不拒识；
- 不重新训练；
- 不根据外部结果返回选择 D1/D2/D3；
- 同时报告 Accuracy、BAcc、AUC、Sensitivity、Specificity、亚型和质量分层；
- workflow 结果若需要展示，必须与基础模型结果完全分表。

## 14. 失败后的明确决策

### 情况 A：H17-0 失败

含义：历史文字概念无法从当前单张照片稳定复现。

行动：

- 不把医生文字变成训练标签；
- 不开展大规模概念模型；
- 只运行 20 例 morphology-only 可见性 mini-pilot；
- mini-pilot 仍失败时，主线转向标准化多视图采集；
- H10 保留为当前单图基础模型上限参照。

### 情况 B：标注一致性或可见性失败

含义：这些形态需要触诊、额外切面或更完整照片，当前图像无法可靠标注。

行动：

- 不要求医生继续扩大标注；
- 制定标准化四视图采集协议：整体外观、主切面、包膜/边界、邻近组织界面；
- 新采集病例形成真正独立的模型开发与泛化队列。

### 情况 C：概念可学习，但风险不提升

含义：可见形态概念真实存在，但不足以决定组织学低危/高危，或标签存在强表型冲突。

行动：

- 不增加风险头容量；
- 不搜索概念权重；
- 分析哪些风险病例拥有相同概念组合；
- 优先补充新视图、触诊/多切面信息或更细粒度病理对应，而不是继续同图调参。

### 情况 D：域内提升，但 source-LODO 失败

含义：模型学到的概念仍受拍摄来源影响。

行动：

- 冻结语义定义，不修改风险阈值；
- 使用多来源未标注图像做来源不变的视觉预训练；
- 优先标准化颜色、拍摄距离、背景和视图角色；
- 在新来源上只做一次锁定验证，不以其结果调参。

### 情况 E：内部和 LODO 通过，严格外部失败

含义：现有三来源仍不足以代表真正外部分布。

行动：

- 如实报告；
- 不用严格外部回训；
- 新增独立医院训练队列后，从头预注册下一轮；
- 不用拒识 workflow 掩盖基础模型外部失败。

## 17. 最终成功标准

H17 只有在以下三层证据同时成立时才算真正提高基础能力：

### 表现层

- 591 例全覆盖直接分类 BAcc 稳定超过 H10；
- 敏感度、特异度和 B1/B2 同时改善；
- 不依赖拒识、复核或阈值搜索。

### 机制层

- 模型能在图像中预测明确形态概念；
- 正确 ROI 和关系比随机对照有效；
- 风险增益依赖形态路径，但不被不完整概念硬限制；
- 收益可以推广到没有形态标注的病例。

### 泛化层

- 多折、多种子稳定；
- 三来源和 source-LODO 不崩溃；
- 最后才在严格外部一次性验证。

如果只能满足表现层而不能满足机制层和泛化层，则不能声称基础视觉能力获得可信提升。

## 19. 一句话决策

下一轮不再问“怎样让模型更重视某个自动显著区域”，而是严格验证：

> 当前照片中是否存在可重复识别的病理形态语义，以及把这些空间锚定语义组合起来后，能否稳定提高全覆盖低危/高危直接分类。

这仍然是提升模型本身能力的主线。若这条链路失败，结论不是再加一个门控或再调一轮参数，而是承认当前单张照片缺少足够监督或信息，转向标准化多视图和新增独立医院数据。
