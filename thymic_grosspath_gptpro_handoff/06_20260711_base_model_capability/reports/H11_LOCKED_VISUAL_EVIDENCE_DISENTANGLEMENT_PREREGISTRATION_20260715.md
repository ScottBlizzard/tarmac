# H11 锁定视觉证据解耦诊断预注册

日期：2026-07-15

## 目的

H10 证明原来的 `easy / boundary / hard` 不是单一视觉难度轴。H11 不训练新模型，也不调阈值，而是对锁定的 H10 模型做反事实推理，区分三种机制：

1. 模型没有稳定找到标本、切面或边界；
2. 模型看到了大体表型，但表型与组织学风险标签相冲突；
3. 单张大体图像本身没有暴露足够证据。

只有第 1 类机制获得预注册意义上的支持，才进入 H12 特征级多分支训练。H11 不能被解释为新的模型性能估计，也不能替代完整 591 例的嵌套交叉验证。

## 锁定资产

- 内部开发队列：591 例，三批数据视为同一个内部队列，仅按六亚型构造五折。

## 原图复现硬闸门

首先重新从 117 张服务器缓存图像提取原始六视图：

`whole, crop, crop_q0, crop_q1, crop_q2, crop_q3`

并使用相同 PE 权重、相同头结构、相同外层折检查点重新推理。只有同时满足以下条件，才运行其他干预：

1. 117 例 × 3 模型的 `0.5` 分类结果与历史 OOF 100% 一致；
2. 新旧 `prob_high` 最大绝对差不超过 `0.005`。

若失败，H11 停止，先排查输入图像、预处理、权重、代码版本或数值精度差异；不得把不可复现结果继续解释为视觉机制。

## 固定反事实视图

所有视图生成规则在看到 H11 结果前固定。视图生成不读取风险标签、亚型、医生文字、历史概率或难度层。

| 变体 | 六个输入槽 | 回答的问题 |
| --- | --- | --- |
| `original_standard6` | 原始锁定六视图 | 复现基线 |
| `tight_standard6` | 先按标本框以 2% 边距紧裁，再生成六视图 | 标本在原图中过小是否是主要瓶颈 |
| `neutral_background_standard6` | 保留自动标本掩膜内像素，将外部背景置为中灰，再生成六视图 | 蓝板、手套、标尺和周边环境是否干扰 |
| `scale_normalized_standard6` | 紧裁标本居中放入中灰正方形，使最长边固定占 84%，再生成六视图 | 病灶尺度/构图是否造成系统性偏差 |
| `evidence_top4` | 原图、紧裁标本、2 个内部异质性窗口、2 个包膜/交界窗口 | 固定象限是否遗漏真正有信息的局部证据 |
| `background_only_standard6` | 标本掩膜内置为中灰，仅保留背景，再生成六视图 | 模型是否仍能从采集背景预测标签；仅作负对照 |

`evidence_top4` 的窗口选择也是无标签固定算法：在紧裁图上用两个窗口尺度和 5×5 网格产生候选；内部窗口按灰度离散度、颜色离散度和局部梯度排序，交界窗口按自动标本掩膜的边界密度与局部异质性排序；同类窗口用 IoU `<=0.35` 的非极大值抑制选两个。候选不足时按固定象限顺序补齐，不允许人工逐病例选 ROI。

## 分析范围

1. **主要净效应范围**：113 例图像同名子集，避免把 4 例本地审计图与服务器模型实际输入图不一致混入机制结论。
2. **可恢复漏读范围**：3 例 `recoverable_visual_model_miss`，全部图像同名。
3. **稳定锚点范围**：图像同名且 H10 三模型均正确的 `M3` 病例。
4. **机制分层范围**：58 例图像同名盲法审计集，按证据充分度和事后 hard 归因报告概率移动。
5. **次要完整范围**：117 例全部报告，但不用于主要通过判定。

每个变体必须作为一个对所有病例完全相同的固定方案评估。禁止根据病例标签选择不同变体，禁止搜索阈值，禁止用多个变体概率加权，禁止只报告获益病例。

## H12 GO / NO-GO 闸门

候选顺序在结果前固定为：

1. `evidence_top4`
2. `scale_normalized_standard6`
3. `neutral_background_standard6`
4. `tight_standard6`

每个候选独立判断。主模型 `internal_risk_balanced` 必须同时满足：

1. 3 例可恢复视觉漏读中至少 2 例变为正确，或其真类概率提高至少 `0.15`；且 3 例真类概率变化中位数至少 `+0.10`。
2. 稳定 `M3` 锚点的正确转错误比例不超过 `5%`。
3. 在 113 例图像同名子集中，`救回数 - 伤害数 >= 3`。
4. B1 和 B2 各自的 `救回数 - 伤害数` 均不得为负。
5. 三个锁定模型在 113 例上的净效应中位数不得为负，任何一个模型的净损失不得超过 3 例。

只有同时通过五条的候选才可进入 H12。若多个候选通过，采用上面的预先固定优先级，而不是事后选择数值最高者。

`background_only_standard6` 不参与 GO 判定。若其主要子集 BAcc `>=0.65` 或 AUC `>=0.70`，记为明显背景捷径信号，后续模型必须加入背景不变性设计和来源外验证，但不能把背景模型并入最终分类器。

## H12 的含义

H11 通过只说明“现有模型的部分错误可由无标签、固定的证据定位方案恢复”，不说明新模型已经提升。H12 才会在完整 591 例、相同外层五折下训练 `whole + specimen + cut/boundary ROI` 特征级多分支模型：

- 分支学习图像内容，不使用置信度门控决定类别；
- 不强制 whole 与 ROI 概率一致；
- 医生概念只用于分层审计和错误解释，不作为部署输入；
- 所有模型选择必须在外层训练折内部完成；
- 最终仍以全覆盖二分类、BAcc、各亚型召回和 source-LODO/严格外部结果判断，而不是放行比例。

## 隐私与存储

- Git 仅提交预注册、代码和聚合报告，不提交病例号、医生原文、图像或逐病例预测。
- 原始图像、反事实图像和特征不从服务器下载，也不长期保存新的密集特征库。
- 服务器仅流式提取每个病例的特征，并保存小型私有概率表和聚合统计。
