# H11 锁定视觉证据解耦诊断结果

日期：2026-07-15

## 最终决策

`H11 NO-GO: CLOSE FIXED ROI/PREPROCESSING BRANCH`

四个预注册固定视觉干预均未通过。紧裁、背景中和、尺度归一化和无标签局部证据窗口都没有稳定救回盲法审计中 3 例“图上已有高危证据但模型漏读”的病例，也没有提高 113 例图像同名子集的全覆盖二分类能力。

这不是执行失败。原始 H10 OOF 预测得到完整复现，六种视图均按锁定规则运行完毕。结果说明：当前最主要问题不是简单的标本过小、蓝板背景、固定象限或自动 crop 方式，而是冻结 PE 表征与当前监督训练没有把可见的大体形态转化成稳定风险语义。

## 锁定执行

- 诊断子集：117 例；A/AB/B1/B2/B3 各 20 例，TC 17 例。
- 与 H10 服务器实际输入图像同名：113/117。
- 三个锁定模型：natural、risk-balanced、subtype-tempered。
- 每例仅使用其 H10 外层留出折模型；固定阈值 `0.5`。
- 六个视图变体，每个变体 `117 × 3 = 351` 条预测，总计 2106 条。
- 无重新训练、无阈值搜索、无病例级变体选择、无概率融合。
- PE 提取和推理总用时：656.6 秒。
- 仅保存小型私有概率表和视图元数据；未保存新密集特征库，未从服务器下载图像或模型。

3 例可恢复视觉漏读在原模型中的真类概率均低于 0.003。四个固定候选下没有一例转为正确，主模型下任何候选的单例真类概率也未超过约 0.021。这些不是阈值附近的小偏差，而是冻结表示和分类头共同形成的强方向性错误。

`evidence_top4` 的无标签窗口选择并未普遍退化为回退象限：117 例的回退窗口中位数为 0，内部和边界候选的中位数均为 2。因此 NO-GO 不能主要归因于候选生成器从未找到局部窗口；更直接的解释是，当前冻结模型没有把这些局部外观解释成正确风险证据。

## 背景负对照

将自动标本掩膜区域置灰、只保留背景后，主模型在 113 例上：

- BAcc `0.443`；
- AUC `0.430`；
- 救回 12、伤害 52，净增益 `-40`；
- 未触发预注册背景捷径信号。

负对照偶然把部分原 hard 推向高危，但这是删除主要图像内容后的分布外失控。它同时破坏了大量原本正确病例，不能解释成“背景专家”，也不能进入任何融合、路由或工作流。

## 机制结论

H11 将下面四个简单解释进一步削弱：

1. **不是只要裁得更紧**：紧裁的三模型净效应全部为负。
2. **不是蓝板背景主导分类**：背景删除几乎摧毁模型，背景单独也没有有效排序能力。
3. **不是统一尺度即可恢复**：尺度归一化伤害最大，说明固定缩放不能补回语义。
4. **不是把固定象限换成高异质/高边界窗口即可**：局部证据窗口没有推动 3 例可恢复 hard 的主模型概率。

当前错误更接近**表征语义和病例级监督不足**：PE-Spatial 的冻结 token 能形成约 0.80 的混合来源边界，但对关键 B1/B2/B3 形态仍会高置信度映射到错误风险方向。重新排列同一冻结 token 的输入方式不能让模型学会此前没有形成的语义关系。

## 下一步约束

关闭当前固定 ROI/预处理 H12，不训练预注册中设想的同一路径 `whole + specimen + cut/boundary ROI` 多分支头。继续重复 crop、尺度、背景或窗口规则只会增加同队列选择偏差。

下一项实验若继续，必须改变冻结表征这一假设，而不是再改置信度或浅层头：优先审计并试运行**现有本地 PE 权重的有限表征适配**，使用严格外层折、固定 0.5 全覆盖分类和 source-LODO 先行；医生概念仍只用于事后审计，不作为训练或部署输入。若有限表征适配不能同时改善 B1 与 B2，则当前固定数据上的模型训练方向应正式结束，转向新增病例、同病例多视图或前瞻性四槽位采集。
