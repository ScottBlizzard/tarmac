# 胸腺大体图像四视图多中心前瞻性采集与封存外测方案（2026-07-13）

## 一、目的与适用范围

本方案用于重新建立胸腺大体图像模型开发的科学有效性，解决当前 591 例历史数据中“单中心、绝大多数病例仅一张图、采集批次与风险特征纠缠”的问题。

当前历史数据仅作为遗留辅助队列，不计入新的多中心、多视图证据。新模型开发必须以本方案的新采集数据为主，严格外测医院在模型冻结前不得参与训练、自监督学习、归一化拟合、阈值选择或方案调优。

## 二、队列结构与最低规模

### 1. 开发队列

- 至少 4 家开发医院；
- 目标约 800 例有最终病理标签的新病例；
- 每家医院均须同时包含低危、高危病例，并尽量覆盖 B1、B2 及其他主要亚型；
- 原医院可以作为开发医院之一，但只能计入按新协议前瞻性采集的病例；
- 不允许单家医院占据绝对多数。

建议的建模型富集目标如下，不代表真实患病率：

| 亚型 | 建议例数 |
|---|---:|
| A | 60 |
| AB | 200 |
| B1 | 140 |
| B2 | 170 |
| B3 | 80 |
| TC | 150 |
| 合计 | 800 |

### 2. 封存外测队列

- 至少 2 家、优选 3 家从未进入开发流程的医院；
- 目标约 600 例新病例；
- 建议低危、高危各约 300 例；
- 建议亚型构成：A 50、AB 150、B1 100、B2 130、B3 60、TC 110；
- 外测病例和医院不得进入任何有监督或无监督训练。

### 3. 大体图像自监督扩展池

只有在获得至少约 5,000 例、20,000 张图、来自 5 家以上开发医院的无标签数据后，才重新考虑实质性的胸腺大体图像自监督预训练。外测医院图像不得进入该池。

## 三、每例四个强制拍摄槽位

每例病例必须按固定顺序采集以下四个槽位，不允许根据主观“最像恶性”来选择是否拍摄：

1. **V1 整体标本总览**：完整标本、固定背景、标尺和色卡同时入镜；
2. **V2 主要完整切面总览**：覆盖主要完整切面，作为预先指定的主单图；
3. **V3 最大存活实性区域近照**：依据面积最大的可见实性区域这一几何规则选取，不依据术后病理或恶性判断；
4. **V4 包膜/肿瘤-邻近组织界面近照**：无可见界面时，按预先规定改拍第二完整切面，并记录替代原因。

V2 必须在病理结果未知时被指定为主单图。后续模型必须进行“同一编码器、同一训练流程、仅 V2”与“全部四视图”的配对比较，以证明增益来自新增视觉信息，而不是置信度门控或后处理调参。

允许保留额外照片，但主要实验输入固定为上述四个槽位。所有重拍、失败图和被质控拒绝的原始文件均须保留。

## 四、拍摄条件

- 统一哑光中性背景；
- 统一规格标尺和色卡，均置于不遮挡标本的位置；
- 尽量使用固定照明，不混用闪光灯与环境光；
- 记录相机或手机机身、镜头、焦距、原始分辨率和文件格式；
- 记录拍摄距离、白平衡、曝光、是否闪光、压缩和导出历史；
- 保留原始分辨率和原始文件，不在医院端美化、锐化、滤镜处理或截图转存；
- 明确记录新鲜或固定状态、标本方向、视图槽位及重拍原因。

不同医院可以使用不同设备，以真实检验泛化，但每家医院内部应固定流程，并完整记录设备变化日期。

## 五、文件命名与去标识化

推荐格式：

`站点代码_研究病例号_视图槽位_重复序号.扩展名`

示例：`S02_C000123_V2_R01.jpg`。

文件名和图像中不得出现姓名、住院号、身份证号、电话号码等直接身份信息。病例映射表由医院数据管理员单独保存，不上传到建模目录。

每个文件生成 SHA-256；上传后再次计算并核对。不得通过聊天软件二次压缩后作为原始数据。

## 六、完整性与质控门槛

满足以下全部条件后，才允许重新启动模型训练：

- 总体至少 95% 病例具备四个完整槽位；
- 每家开发医院四视图完整率至少 90%；
- 所有符合纳入条件的病例均进入清单，包括采集失败病例；
- 缺失槽位及原因在开放病理标签前完成记录；
- 预先实现固定的缺失槽位掩码，使模型对所有入组病例均能输出结果；
- 不得回顾性删除低质量病例以人为提高指标。

自动质控只用于提示重拍和分析采集问题，不能作为选择性放行病例的模型性能替代。

## 七、必须记录但不输入主模型的元数据

- 医院、采集日期或时期；
- 操作者角色；
- 设备机身、镜头、焦距、原始分辨率；
- 文件格式和压缩历史；
- 光源、闪光、白平衡、曝光；
- 拍摄距离；
- 背景、标尺、色卡是否存在；
- 新鲜或固定状态；
- 标本方向和视图槽位；
- 拍摄张数、重拍与失败原因；
- 裁剪、缩放、增强、导出和去标识化历史；
- 缺失视图原因。

这些字段用于质控、分层报告和偏倚审计，不作为主图像分类器输入。

## 八、病理标签与人工核验

- 最终 WHO 亚型采用统一诊断规则；
- 混合型、B1/B2 边界病例优先进行双人或中心复核；
- 保存原始诊断、复核诊断、是否一致、分歧原因和最终裁决；
- 医生描述字段属于标签或审计变量，不直接输入主图像模型；
- 在图像清单、采集元数据和缺失原因封存前，不向建模团队释放最终标签。

## 九、开发与封存外测治理

### 开发阶段

1. 先完成医院、槽位、设备和完整性审计；
2. 再开放开发医院病理标签；
3. 患者级划分，禁止同一病例跨训练与验证；
4. 以医院留一验证作为主要内部泛化检查；
5. 六分类为重要次要任务，Task7 低危/高危仍为主要任务；
6. 主模型直接输出，默认阈值 0.5，不以选择性放行覆盖全量性能。

### 外测阶段

1. 外测医院在模型系列、预处理、权重、代码哈希、视图处理和阈值全部冻结后才提供测试图像；
2. 另设不属于正式外测的技术小包，仅验证传输、解码和文件结构；
3. 禁止在正式外测图像上进行自监督、归一化拟合、测试时适配、阈值选择或模型选择；
4. 对全部病例生成 100% 覆盖的预测、文件哈希和运行日志；
5. 预测文件先交独立保管人封存，再与病理标签合并；
6. 分医院报告，同时报告合并结果；
7. 必须配对比较 V2 单图模型与四视图模型。

## 十、主要分析与报告

主要报告：

- 全量平衡准确率、AUC、敏感度、特异度；
- 每家医院独立结果；
- A、AB、B1、B2、B3、TC 每亚型结果；
- B1/B2 边界表现；
- V2 单图与四视图的病例级配对差异；
- 医院×风险分层的配对 bootstrap 置信区间；
- 失败病例、缺失视图和低质量病例的全覆盖分析。

模型若需要人工复核，可作为次要工作流分析，但不能替代全量直接分类结果，也不能仅靠域内置信度门控形成主要结论。

## 十一、医院启动前检查清单

- [ ] 确定开发医院与封存外测医院名单，角色不可混用；
- [ ] 各医院完成四槽位拍摄培训；
- [ ] 标尺、色卡、背景和照明方案到位；
- [ ] 完成病例号、槽位号、重复号命名测试；
- [ ] 完成 SHA-256 和安全上传测试；
- [ ] 完成采集元数据表和缺失原因表；
- [ ] 明确标签负责人、双人复核规则和裁决流程；
- [ ] 指定外测预测文件独立保管人；
- [ ] 完成开发医院小规模流程试运行；
- [ ] 在任何模型训练前签署并冻结本方案版本。
